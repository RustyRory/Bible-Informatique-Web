1. Concepts avancés de commutation

1.1 VLANs (Virtual Local Area Networks) et leur configuration

Qu’est-ce qu’un VLAN ?

  • Un VLAN (Virtual Local Area Network) est une segmentation logique d’un réseau local physique.
  • Les VLAN permettent de regrouper les ports des commutateurs dans des segments réseau indépendants, même si ces ports sont physiquement sur différents commutateurs.

Avantages des VLANs :

  1. Isolation des réseaux :
    • Chaque VLAN agit comme un réseau indépendant.
    • Réduit le trafic inutile en limitant les broadcast à un VLAN.
  1. Amélioration de la sécurité :
    • Les VLAN isolent les utilisateurs ou les services sensibles.
    • Exemple : Un VLAN pour les serveurs, un pour les invités.
  1. Facilité d’administration :
    • Les VLAN permettent de gérer les groupes logiques sans changer la topologie physique.

1.2 Protocole STP (Spanning Tree Protocol) et ses variantes

Qu’est-ce que le STP ?

  • Le protocole STP (Spanning Tree Protocol) empêche les boucles réseau dans une architecture de commutateurs redondante.
  • Fonctionnement :
    • STP désactive automatiquement les ports redondants pour créer une topologie arborescente sans boucle.

Étapes du STP :

  1. Élection du Root Bridge :
    • Chaque commutateur compare son identifiant de pont (Bridge ID) pour élire le Root Bridge (commutateur central de la topologie).
  1. Sélection des ports Root :
    • Chaque commutateur identifie son port le plus rapide pour atteindre le Root Bridge.
  1. Désactivation des ports redondants :
    • Les ports non essentiels sont mis en état bloqué pour éviter les boucles.

Variantes de STP :

  1. RSTP (Rapid Spanning Tree Protocol, 802.1w) :
    • Amélioration de STP classique avec une convergence plus rapide.
  1. MSTP (Multiple Spanning Tree Protocol, 802.1s) :
    • Permet la gestion simultanée de plusieurs instances STP pour différents VLANs.
  1. PVST+ (Per VLAN Spanning Tree) :
    • Implémentation Cisco qui gère une instance STP pour chaque VLAN.

1.3 EtherChannel et agrégation de liens

Qu’est-ce qu’EtherChannel ?

  • EtherChannel regroupe plusieurs liaisons physiques entre deux commutateurs pour en faire une seule liaison logique.
  • Avantages :
    1. Augmente la bande passante disponible.
    1. Fournit une redondance (les liaisons restantes prennent le relais en cas de panne).
    1. Réduit la complexité du STP (les liens agrégés sont vus comme un seul lien logique).

Protocole de négociation d’EtherChannel :

  1. PAgP (Port Aggregation Protocol) :
    • Protocole propriétaire Cisco.
  1. LACP (Link Aggregation Control Protocol) :
    • Standard IEEE (802.3ad), interopérable avec des équipements non-Cisco.

2. Routage inter-VLAN

2.1 Configuration du routage entre VLANs

Principe

Chaque VLAN correspond à un sous-réseau. Pour que les hôtes de différents VLANs puissent communiquer, leur trafic doit passer par une interface de routage (routeur ou commutateur de couche 3).

Conditions nécessaires :

  1. Chaque VLAN doit avoir une adresse IP de passerelle attribuée.
  1. Les appareils connectés doivent configurer cette passerelle comme route par défaut.

2.2 Router-on-a-stick et sous-interfaces

Qu’est-ce que "Router-on-a-stick" ?

C’est une méthode classique où un routeur unique est utilisé pour router le trafic entre plusieurs VLANs en utilisant une seule interface physique. Cette interface est subdivisée en plusieurs sous-interfaces, chacune configurée avec une adresse IP et associée à un VLAN.

Fonctionnement de "Router-on-a-stick" :

  1. La liaison entre le routeur et le commutateur est configurée comme un lien trunk (802.1Q).
  1. Les sous-interfaces sur le routeur prennent en charge les VLANs et assurent le routage entre eux.

Limites du Router-on-a-stick :

  1. Performances limitées : Un seul lien physique pour tout le trafic inter-VLAN.
  1. Risques de congestion : Les environnements à forte charge nécessitent des solutions plus robustes comme les commutateurs de couche 3.

2.3 Commutateurs de couche 3

Qu’est-ce qu’un commutateur de couche 3 ?

Un commutateur de couche 3 combine les fonctionnalités d’un commutateur et d’un routeur :

  • Fonctionne comme un commutateur pour les communications au sein d’un VLAN.
  • Route le trafic entre VLANs directement, sans besoin d’un routeur externe.

Avantages par rapport à Router-on-a-stick :

  1. Performance améliorée : Le routage est effectué directement dans le matériel, évitant les goulots d’étranglement.
  1. Simplification de l’architecture : Pas besoin d’utiliser un routeur séparé pour le routage inter-VLAN.
  1. Évolutivité : Idéal pour les réseaux de grande taille nécessitant un trafic inter-VLAN important.

3. Protocoles de routage dynamique

3.1 OSPF (Open Shortest Path First) pour IPv4 et IPv6

Qu’est-ce qu’OSPF ?

  • OSPF (Open Shortest Path First) est un protocole de routage dynamique à état de liens.
  • Standard ouvert, il est largement utilisé pour les réseaux de grande taille.
  • OSPF calcule les routes les plus courtes en utilisant l’algorithme SPF (Shortest Path First) basé sur le coût de chaque lien.

Caractéristiques principales d’OSPF :

  1. Hiérarchique :
    • Utilise des zones pour optimiser les mises à jour (Zone 0 ou backbone, zones non-backbone).
  1. Convergence rapide :
    • Réagit rapidement aux changements du réseau.
  1. Support d’IPv4 et IPv6 :
    • Utilise OSPFv2 pour IPv4 et OSPFv3 pour IPv6.
  1. Authentification :
    • Permet de sécuriser les mises à jour de routage avec des mots de passe ou des signatures.

3.2 EIGRP (Enhanced Interior Gateway Routing Protocol)

Qu’est-ce qu’EIGRP ?

  • EIGRP est un protocole de routage dynamique à vecteur de distance avancé, développé par Cisco.
  • Combine les avantages des protocoles à vecteur de distance (simplicité) et des protocoles à état de liens (rapidité et précision).

Caractéristiques principales d’EIGRP :

  1. Algorithme DUAL (Diffusing Update Algorithm) :
    • Garantit des routes sans boucle et une convergence rapide.
  1. Simplicité d’administration :
    • Facilite la configuration avec des fonctionnalités avancées.
  1. Support IPv4 et IPv6 :
    • Fonctionne sur les deux types de réseaux.
  1. Protocoles multiples :
    • Prend en charge plusieurs protocoles comme IP, IPv6 et AppleTalk.

3.3 Comparaison : Protocole à vecteur de distance vs Protocole à état de liens

Critère Vecteur de distance État de liens
Exemples RIP, EIGRP OSPF, IS-IS
Algorithme Simple : Distance au prochain saut Complexe : Vue complète du réseau
Convergence Lente Rapide
Utilisation Petits réseaux Réseaux moyens à grands
Simplicité Facile à configurer Plus complexe

Vecteur de distance :

  • Les routeurs échangent des informations de routage limitées (distance et saut suivant).
  • Converge moins rapidement car chaque routeur dépend des mises à jour des voisins.

État de liens :

  • Les routeurs maintiennent une vue complète de la topologie réseau.
  • Plus précis et rapide, mais consomme davantage de ressources.

4.2 VPNs (Virtual Private Networks)

Qu’est-ce qu’un VPN ?

Un VPN crée une connexion sécurisée sur un réseau non sécurisé (par exemple, Internet). Il chiffre les données et assure leur confidentialité et intégrité.

Types de VPN :

  1. VPN Site-à-Site :
    • Connecte deux réseaux LAN via une connexion sécurisée.
    • Utilisé pour interconnecter des succursales ou des datacenters.
  1. VPN Accès à distance :
    • Permet aux utilisateurs distants de se connecter à un réseau centralisé.
    • Exemples : VPN SSL, VPN IPsec.

Protocoles VPN courants :

  1. IPsec (Internet Protocol Security) :
    • Fournit un chiffrement et une authentification robustes.
  1. SSL/TLS :
    • Utilisé pour sécuriser les VPN d’accès à distance.
  1. GRE (Generic Routing Encapsulation) :
    • Encapusle les protocoles pour créer des tunnels, souvent utilisé avec IPsec.

5. Qualité de service (QoS)

5.1 Principes de base de la QoS

Définition :

  • La QoS contrôle la façon dont les paquets de données sont transmis sur un réseau afin de répondre à des exigences spécifiques en termes de :
    • Bande passante : Assurer suffisamment de débit pour le trafic critique.
    • Latence : Réduire les délais pour les applications temps réel.
    • Gigue (jitter) : Minimiser la variation du délai pour les flux audio/vidéo.
    • Pertes : Réduire les paquets perdus dans le réseau.

Objectifs principaux :

  1. Priorisation : Offrir un traitement préférentiel au trafic critique.
  1. Allocation de bande passante : Réserver des ressources pour certaines applications.
  1. Réduction de la congestion : Gérer les surcharges pour éviter la dégradation des performances.

Modèles de QoS :

  1. Best-Effort :
    • Aucun mécanisme QoS n’est appliqué, tous les paquets sont traités équitablement.
    • Adapté pour des réseaux sans trafic critique.
  1. Integrated Services (IntServ) :
    • Réserve explicitement des ressources pour chaque flux.
    • Efficace mais complexe à mettre en œuvre.
  1. Differentiated Services (DiffServ) :
    • Classe les paquets dans des catégories (classes) et applique des politiques spécifiques à chaque classe.

5.2 Méthodes de classification et de marquage du trafic

Classification :

  • Processus de catégorisation des paquets en fonction de certains critères, tels que :
    • Adresse IP source/destination.
    • Port TCP/UDP.
    • Type de protocole (HTTP, VoIP, etc.).

Marquage :

  • Les paquets sont marqués pour indiquer leur priorité ou leur classe de service. Les marques courantes incluent :
    1. DSCP (Differentiated Services Code Point) :
      • Champs de 6 bits dans l'en-tête IP pour définir la classe de service.
    1. CoS (Class of Service) :
      • Utilisé dans les en-têtes Ethernet VLAN pour marquer les priorités au niveau de la couche 2.

5.3 Gestion de la congestion et de la bande passante

Problème de congestion :

  • La congestion survient lorsque la demande en bande passante dépasse la capacité d’un lien ou d’un équipement réseau, ce qui entraîne des retards et des pertes.

Techniques de gestion de la congestion :

  1. Queuing (Mise en file d’attente) :
    • Organise les paquets dans des files d'attente en fonction de leur priorité.
    • Méthodes courantes :
      • FIFO (First-In-First-Out) : Simple, mais ne traite pas les priorités.
      • Priority Queuing (PQ) : Les files de priorité élevée sont traitées en premier.
      • Weighted Fair Queuing (WFQ) : Répartit équitablement la bande passante entre les flux.
  1. Policing et Shaping :
    • Policing : Limite la bande passante maximale pour un trafic donné. Les paquets excédentaires sont supprimés.
    • Shaping : Lisse le trafic pour éviter les rafales et respecte une bande passante définie.
  1. RED (Random Early Detection) :
    • Réduit la congestion en supprimant de manière aléatoire les paquets avant que la file d’attente ne soit pleine.

6. Sécurité des réseaux

6.1 ACLs (Access Control Lists) standard et étendues

Qu'est-ce qu'une ACL ?

  • Une Access Control List (ACL) est un ensemble de règles appliquées sur un routeur ou un commutateur pour contrôler le trafic réseau en autorisant ou refusant des paquets en fonction de critères spécifiques.

6.3 Principes de base des pare-feux et de la détection d'intrusion

Pare-feux :

  1. Fonctionnalités clés :
    • Filtrage des paquets : Basé sur l’adresse IP, les ports et les protocoles.
    • Inspection d'état (stateful inspection) : Suit les connexions actives pour une gestion dynamique.
    • Proxy : Agit comme intermédiaire pour analyser les requêtes avant leur transmission.

Détection et prévention d'intrusion (IDS/IPS) :

    • IDS (Intrusion Detection System) : Détecte les activités malveillantes et envoie des alertes.
    • IPS (Intrusion Prevention System) : Bloque activement les menaces identifiées.
  1. Fonctionnement :

7. Dépannage réseau

7.1 Méthodologie de dépannage structurée

7.1.1. Identifier le problème :

  • Symptômes : Recueillir des informations auprès des utilisateurs ou via des outils réseau.
    • Exemple : Absence de connectivité, lenteur, déconnexion intermittente.
  • Définir les limites : Identifier les zones impactées (un utilisateur, un VLAN, tout le réseau).

7.1.2. Formuler une hypothèse :

  • Analyser les symptômes pour déterminer la cause probable.
    • Exemples : Un problème matériel, une configuration erronée, ou une surcharge réseau.

7.1.3. Tester l'hypothèse :

  • Effectuer des actions correctives ou des tests ciblés pour valider l’hypothèse.
    • Exemple : Pinger une adresse IP ou inspecter la table ARP.

7.1.4. Implémenter une solution :

  • Appliquer des correctifs pour résoudre le problème.

7.1.5. Vérification :

  • Confirmer que le problème est résolu et que le réseau fonctionne normalement.

7.1.6. Documenter :

  • Enregistrer les détails du problème, la solution appliquée et les leçons apprises pour référence future.

8. Automatisation et programmabilité des réseaux

8.1 Introduction aux concepts de SDN (Software-Defined Networking)

Définition du SDN :

  • Software-Defined Networking (SDN) est une approche de l'architecture réseau qui sépare le plan de contrôle (qui prend des décisions sur le flux de données) et le plan de données (qui acheminent les paquets). Cela permet une gestion plus centralisée et dynamique du réseau.

Principes fondamentaux du SDN :

  1. Séparation des plans de contrôle et de données :
    • Le plan de contrôle gère la logique de routage et de commutation (par exemple, les décisions de transmission de paquets), tandis que le plan de données gère le transfert réel des paquets via les équipements réseau.
    • Cela permet une plus grande flexibilité, car les décisions peuvent être prises centralement, sans avoir besoin de configurations manuelles sur chaque périphérique.
  1. Contrôleur SDN :
    • Le contrôleur SDN est un composant central qui gère l’ensemble du réseau. Il communique avec les équipements réseau via un protocole de contrôle pour mettre à jour les tables de routage et appliquer des politiques réseau.
    • Exemple de contrôleur SDN : OpenFlow, Cisco ACI, ONOS.
  1. Programmes et applications SDN :
    • Les applications SDN peuvent être utilisées pour automatiser des tâches telles que la gestion de la bande passante, la configuration des VLANs, la sécurité réseau, et plus encore. Ces applications communiquent avec le contrôleur SDN pour ajuster les configurations réseau en temps réel.

Avantages du SDN :

  • Centralisation : Facilite la gestion et la configuration du réseau à partir d’un seul point central.
  • Agilité : Permet des modifications rapides et dynamiques de la topologie et des règles de routage.
  • Automatisation : Réduit la nécessité d'interventions manuelles répétitives.

8.2 APIs et protocoles de gestion de réseau (NETCONF, RESTCONF)

APIs (Interfaces de Programmation d'Applications) :

  • Les APIs permettent aux applications et aux services externes de communiquer avec les équipements réseau pour leur configuration, leur gestion et leur surveillance.

Protocoles de gestion de réseau :

  1. NETCONF (Network Configuration Protocol) :
    • NETCONF est un protocole de gestion de réseau basé sur XML qui permet de configurer, surveiller et contrôler les périphériques réseau.
    • Fonctionnalités :
      • Il permet une gestion structurée des configurations, avec un contrôle de version et une capacité de gestion des erreurs.
      • Il est souvent utilisé dans des environnements où une gestion robuste de la configuration est nécessaire.
    • Exemple d’utilisation : Configuration dynamique d'un routeur ou d'un switch via un contrôleur SDN ou un serveur d'automatisation.
  1. RESTCONF :
    • RESTCONF est une extension du protocole REST (Representational State Transfer) qui est spécifiquement conçu pour la gestion des réseaux.
    • Fonctionnalités :
      • RESTCONF fournit une API web basée sur HTTP qui est beaucoup plus simple à intégrer et à utiliser que NETCONF. Il est basé sur des services RESTful et est souvent utilisé pour interagir avec des équipements réseau via des requêtes HTTP.
      • Idéal pour une communication plus légère et accessible avec des systèmes de gestion de réseau via des applications externes ou des outils d’automatisation comme Ansible.
    • Exemple d’utilisation : Contrôler et configurer des périphériques réseau via des appels HTTP (GET, POST, PUT).

Exemple d'intégration avec SDN :

Les contrôleurs SDN peuvent utiliser NETCONF ou RESTCONF pour envoyer des configurations directement aux périphériques réseau (commutateurs, routeurs), permettant une gestion et un contrôle dynamique du réseau via des interfaces programmables.

8.3 Notions de base sur les outils d'automatisation (ex : Ansible, Python)

Outils d'automatisation réseau :

  1. Ansible :
    • Ansible est un outil d'automatisation de configuration et de gestion des systèmes et des réseaux. Il est simple à utiliser et basé sur des playbooks (scripts de configuration écrits en YAML).
    • Avantages :
      • Pas besoin d’agents : Ansible fonctionne en utilisant SSH pour se connecter aux périphériques.
      • Idéal pour des tâches répétitives telles que la configuration des périphériques réseau, la gestion des VLANs, l'application de politiques de sécurité, etc.
    • Exemple de playbook Ansible pour configurer un VLAN : 
      - name: Configure VLAN on a switch
  hosts: switches
  tasks:
    - name: Create VLAN 10
      ios_vlan:
        vlan_id: 10
        name: "VLAN_10"
        state: present
  1. Python :
    • Python est un langage de programmation très populaire dans le domaine de l'automatisation réseau. Il est utilisé pour écrire des scripts qui interagissent avec des équipements réseau via des API RESTful, NETCONF, ou des modules spécifiques comme PySNMP.
    • Exemple de script Python pour configurer un switch via RESTCONF : 
      import requests
url = "https://<ip_address>/restconf/data/native/interface/Vlan1"
headers = {"Content-Type": "application/yang.data+json"}
data = {
  "name": "Vlan1",
  "ip": {"address": {"primary": {"address": "192.168.1.1", "mask": "255.255.255.0"}}}
}
response = requests.put(url, json=data, headers=headers, auth=('admin', 'password'))
print(response.status_code, response.text)
  1. Avantages de l'automatisation :
    • Consistance : Réduit les erreurs humaines en appliquant les mêmes configurations de manière répétée.
    • Efficacité : Automatise les tâches répétitives, permettant de gagner du temps et d'augmenter la productivité.
    • Scalabilité : Permet de gérer facilement un grand nombre de périphériques et de configurations.
