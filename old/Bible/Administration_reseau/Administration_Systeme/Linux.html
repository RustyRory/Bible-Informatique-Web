1. Introduction à Linux

1.1. Histoire et philosophie de Linux

Origines de Linux

  • Linux a été créé en 1991 par Linus Torvalds, alors étudiant en informatique, comme une alternative libre et gratuite aux systèmes UNIX propriétaires.
  • Le projet s’est inspiré de Minix, un système d’exploitation éducatif basé sur UNIX, développé par Andrew S. Tanenbaum.
  • Le noyau Linux, combiné avec les outils GNU développés par la Free Software Foundation (FSF) dirigée par Richard Stallman, forme ce qu’on appelle communément un système GNU/Linux.

Philosophie et principes de Linux

  • Logiciel libre : Distribué sous licence GPL (GNU General Public License), Linux permet à quiconque de l'utiliser, le modifier, et le redistribuer.
  • Modularité : Linux est un système modulaire où chaque composant peut être indépendant et remplacé ou amélioré.
  • Communauté et collaboration : Développé de manière communautaire, Linux repose sur les contributions de milliers de développeurs à travers le monde.
  • Performance et stabilité : Linux est reconnu pour sa robustesse et son efficacité, en particulier dans les environnements serveurs et embarqués.

1.2. Comprendre les distributions Linux

Qu'est-ce qu'une distribution Linux ?

  • Une distribution Linux (ou "distro") est un ensemble qui combine le noyau Linux avec des logiciels, des outils, et des gestionnaires de paquets pour former un système d'exploitation complet.
  • Chaque distribution cible un public ou un usage spécifique (desktop, serveur, embarqué).

Les principales familles de distributions

  1. Debian :
    • Distribution très stable, privilégiée pour les serveurs.
    • Utilise le gestionnaire de paquets APT.
    • Exemples de dérivés : Ubuntu, Kali Linux.
  1. Red Hat :
    • Ciblée pour les entreprises avec un support professionnel.
    • Utilise les paquets RPM et le gestionnaire yum/dnf.
    • Exemples de dérivés : CentOS, Fedora.
  1. Ubuntu :
    • Basée sur Debian, populaire pour les postes de travail et serveurs.
    • Conviviale pour les débutants avec une large communauté.
  1. CentOS/AlmaLinux/Rocky Linux :
    • Versions gratuites de Red Hat Enterprise Linux (RHEL).
    • Très utilisées dans les environnements professionnels.
  1. Arch Linux :
    • Distribution légère et personnalisable, orientée vers les utilisateurs avancés.
    • Utilise le gestionnaire de paquets pacman.
  1. SUSE/OpenSUSE :
    • Distribution stable et polyvalente, utilisée dans les entreprises.
    • Utilise le gestionnaire de paquets zypper.

Choisir une distribution

  • Débutants : Ubuntu, Linux Mint.
  • Serveurs professionnels : RHEL, CentOS, Debian.
  • Personnalisation : Arch Linux, Gentoo.
  • Sécurité : Kali Linux, Parrot OS.

1.3. Concepts de base : noyau, shell, système de fichiers

Le noyau

  • Définition : Le noyau est le cœur du système d'exploitation Linux. Il agit comme un intermédiaire entre le matériel et les logiciels.
  • Fonctions principales :
    • Gestion des processus : Attribution des ressources et planification.
    • Gestion de la mémoire : Allocation de la mémoire aux programmes.
    • Gestion des périphériques : Interaction avec les pilotes.
    • Sécurité : Contrôle des accès au système.
  • Types de noyaux :
    • Monolithique (comme Linux) : Toutes les fonctions principales sont dans un seul noyau.
    • Micro-noyau (comme Minix) : Les fonctions sont déléguées à des processus externes pour plus de modularité.

Le shell

  • Définition : Le shell est l'interface utilisateur qui interprète les commandes saisies par l'utilisateur et les transmet au système.
  • Types de shell :
    • Bash (Bourne Again Shell) : Shell par défaut dans de nombreuses distributions.
    • Zsh : Similaire à Bash, avec des fonctionnalités avancées.
    • Fish : Shell interactif moderne avec complétion automatique.
    • Dash : Shell minimaliste, souvent utilisé pour les scripts système.
  • Commandes courantes :
    • ls : Liste les fichiers.
    • cd : Change de répertoire.
    • pwd : Affiche le répertoire courant.
    • man : Affiche les manuels des commandes.

Le système de fichiers Linux

  • Organisation : Linux utilise un système hiérarchique où tout est considéré comme un fichier, y compris les périphériques.
    • Racine (/) : Point de départ de tous les fichiers.
    • Répertoires importants :
      • /bin : Commandes essentielles.
      • /etc : Fichiers de configuration.
      • /var : Données variables (logs, cache).
      • /home : Répertoires des utilisateurs.
      • /tmp : Fichiers temporaires.
      • /dev : Fichiers de périphériques.
  • Systèmes de fichiers supportés :
    • ext4 : Système de fichiers par défaut dans de nombreuses distributions.
    • XFS : Conçu pour les grandes bases de données.
    • Btrfs : Pour des snapshots et des fonctionnalités avancées.
  • Commandes de gestion des fichiers :
    • cp : Copier des fichiers.
    • mv : Déplacer ou renommer des fichiers.
    • rm : Supprimer des fichiers.
    • df : Vérifier l’espace disque.
    • du : Vérifier la taille des fichiers/répertoires.
2. Installation et configuration de base

2.1. Installation de Linux : partitions, systèmes de fichiers (ext4, xfs, etc.)

Processus d'installation de Linux

  1. Téléchargement et création du support d'installation :
    • Téléchargez l'image ISO de la distribution choisie.
    • Utilisez un outil comme Rufus ou Etcher pour créer une clé USB amorçable.
  1. Démarrage depuis le support :
    • Configurez le BIOS/UEFI pour démarrer à partir du support d'installation.
  1. Sélection des options d'installation :
    • Choisissez la langue, le clavier, et d'autres paramètres régionaux.
    • Optez pour une installation graphique ou en mode texte selon les besoins.

Partitionnement du disque

  • Qu’est-ce qu’une partition ?
    • Une partition est une division logique d’un disque dur utilisée pour organiser les données.
  • Types de partitions :
    • / (racine) : Contient tout le système d'exploitation.
    • /home : Stocke les données des utilisateurs.
    • /swap : Espace pour la mémoire virtuelle.
    • /boot : Contient les fichiers nécessaires au démarrage.
    • /var et /tmp : Pour des besoins spécifiques comme les logs et fichiers temporaires.
  • Schéma recommandé :
    • Système personnel : / et /home.
    • Serveurs : /, /var, /tmp, /home, /boot.

Systèmes de fichiers

  • ext4 : Standard pour de nombreuses distributions. Rapide et fiable.
  • xfs : Idéal pour les systèmes exigeant de grandes bases de données.
  • btrfs : Système de fichiers moderne pour les snapshots et la gestion avancée.
  • vfat/ntfs : Utilisés pour des compatibilités spécifiques avec Windows.

Outils de partitionnement

  • GParted : Interface graphique pour le partitionnement.
  • fdisk ou parted : Outils en ligne de commande.

2.2. Configuration de l'environnement initial

Nom d'hôte (hostname)

  • Qu’est-ce que le hostname ?
    • Nom unique d’un système sur le réseau.
  • Configuration :
    • Commande temporaire : hostnamectl set-hostname <nom_hôte>.
    • Fichier permanent : /etc/hostname et /etc/hosts.

Gestion des utilisateurs

  • Création d'un utilisateur :
    • Commande : useradd -m -s /bin/bash <nom_utilisateur>.
    • Attribuer un mot de passe : passwd <nom_utilisateur>.
  • Gestion des groupes :
    • Ajouter un utilisateur à un groupe : usermod -aG <groupe> <utilisateur>.
  • Bonnes pratiques :
    • Évitez d'utiliser root pour des tâches courantes.
    • Configurez un utilisateur avec des privilèges sudo (usermod -aG sudo <utilisateur>).

Locales et fuseaux horaires

  • Configuration des locales :
    • Définissez la langue et le format régional : dpkg-reconfigure locales ou locale-gen.
  • Configuration du fuseau horaire :
    • Affichez les fuseaux disponibles : timedatectl list-timezones.
    • Définissez un fuseau : timedatectl set-timezone <zone/ville>.

2.3. Gestion des gestionnaires de paquets

Qu’est-ce qu’un gestionnaire de paquets ?

  • Un gestionnaire de paquets facilite l’installation, la mise à jour et la suppression des logiciels dans Linux.
  • Il fonctionne en téléchargeant les paquets à partir de dépôts centralisés.

Principaux gestionnaires de paquets

  1. APT (Advanced Package Tool) :
    • Utilisé par les distributions basées sur Debian (Ubuntu, Linux Mint).
    • Commandes courantes :
      • Mise à jour de la liste des paquets : sudo apt update.
      • Installation : sudo apt install <paquet>.
      • Suppression : sudo apt remove <paquet>.
      • Mise à niveau du système : sudo apt upgrade.
  1. YUM/DNF :
    • Utilisé par Red Hat, CentOS, Fedora.
    • Commandes courantes (DNF remplace YUM dans les distributions modernes) :
      • Mise à jour de la liste des paquets : sudo dnf update.
      • Installation : sudo dnf install <paquet>.
      • Suppression : sudo dnf remove <paquet>.
  1. Zypper :
    • Utilisé par SUSE et openSUSE.
    • Commandes courantes :
      • Mise à jour des dépôts : sudo zypper refresh.
      • Installation : sudo zypper install <paquet>.
      • Suppression : sudo zypper remove <paquet>.
  1. Pacman :
    • Gestionnaire de paquets d’Arch Linux.
    • Commandes courantes :
      • Synchronisation des paquets : sudo pacman -Syu.
      • Installation : sudo pacman -S <paquet>.
      • Suppression : sudo pacman -R <paquet>.

Gestion des dépôts :

  • Les distributions utilisent des dépôts centralisés pour garantir l’intégrité et la sécurité.
  • Ajout d’un dépôt tiers :
    • APT : Ajoutez un dépôt avec add-apt-repository.
    • DNF/Zypper : Ajoutez manuellement une URL de dépôt.

Bonnes pratiques :

  • Gardez vos paquets à jour pour des raisons de sécurité.
  • Évitez de mélanger des dépôts incompatibles.
  • Supprimez les paquets inutilisés pour économiser de l’espace disque.
3. Commandes de base Linux

3.1. Navigation dans le système de fichiers

Commandes essentielles

  1. cd (Change Directory) :
    • Change le répertoire courant.
    • Exemples :
      • Aller au répertoire /home : cd /home.
      • Revenir au répertoire précédent : cd -.
      • Aller au répertoire utilisateur : cd ~.
  1. ls (List) :
    • Affiche le contenu d’un répertoire.
    • Options courantes :
      • Liste détaillée : ls -l.
      • Affiche les fichiers cachés : ls -a.
      • Tri par taille : ls -lS.
  1. pwd (Print Working Directory) :
    • Affiche le chemin absolu du répertoire courant.
    • Exemples : pwd.
  1. tree :
    • Affiche une vue hiérarchique des fichiers.
    • Exemple : tree /etc.

3.2. Manipulation des fichiers et répertoires

Commandes principales

  1. cp (Copy) :
    • Copie des fichiers ou des répertoires.
    • Exemple : cp fichier1 fichier2.
    • Copier un répertoire : cp -r dossier1 dossier2.
  1. mv (Move) :
    • Déplace ou renomme un fichier ou un répertoire.
    • Exemple pour déplacer : mv fichier1 /home/user/.
    • Exemple pour renommer : mv ancien_nom nouveau_nom.
  1. rm (Remove) :
    • Supprime des fichiers ou des répertoires.
    • Supprimer un fichier : rm fichier1.
    • Supprimer un répertoire et son contenu : rm -r dossier.
  1. mkdir (Make Directory) :
    • Crée un nouveau répertoire.
    • Exemple : mkdir nouveau_dossier.
    • Créer plusieurs répertoires imbriqués : mkdir -p parent/enfant.

3.3. Gestion des utilisateurs et permissions

Permissions

  1. chmod :
    • Modifie les permissions des fichiers ou répertoires.
    • Exemple :
      • Donner des permissions de lecture, écriture et exécution : chmod 755 fichier.
      • Permissions symboliques : chmod u+rwx fichier.
  1. chown :
    • Change le propriétaire ou le groupe d’un fichier.
    • Exemple : chown utilisateur fichier.
    • Pour changer le groupe aussi : chown utilisateur:groupe fichier.
  1. umask :
    • Définit les permissions par défaut pour les nouveaux fichiers.
    • Exemple : umask 022 pour que les nouveaux fichiers aient des permissions 755.

Gestion des utilisateurs

  • Ajouter un utilisateur : useradd -m utilisateur.
  • Supprimer un utilisateur : userdel utilisateur.

3.4. Gestion des processus

Commandes essentielles

  1. ps (Process Status) :
    • Liste les processus en cours d'exécution.
    • Exemple : ps aux pour une vue complète.
  1. top :
    • Affiche en temps réel les processus les plus actifs.
    • Alternative graphique : htop.
  1. kill :
    • Termine un processus en utilisant son PID.
    • Exemple : kill 12345.
  1. jobs :
    • Affiche les tâches en arrière-plan associées au terminal.
    • Exemple : jobs.
  1. nohup :
    • Exécute une commande même après la fermeture de la session.
    • Exemple : nohup commande &.

3.5. Compression et archivage

Commandes de base

  1. tar :
    • Archive des fichiers.
    • Exemple : tar -cvf archive.tar dossier.
    • Extraire une archive : tar -xvf archive.tar.
  1. gzip / bzip2 :
    • Compression des fichiers.
    • Exemple : gzip fichier ou bzip2 fichier.
    • Décompression : gunzip fichier.gz ou bunzip2 fichier.bz2.
  1. zip / unzip :
    • Créer une archive compressée : zip archive.zip fichier1 fichier2.
    • Extraire une archive : unzip archive.zip.

3.6. Réseaux de base

Commandes utiles

  1. ping :
    • Vérifie la connectivité réseau avec une adresse ou un domaine.
    • Exemple : ping google.com.
  1. netstat :
    • Affiche les connexions réseau.
    • Exemple : netstat -tuln.
  1. curl :
    • Récupère des données d’une URL.
    • Exemple : curl https://example.com.
  1. wget :
    • Télécharge des fichiers depuis une URL.
    • Exemple : wget https://example.com/fichier.
  1. traceroute :
    • Affiche le chemin suivi par les paquets vers une destination.
    • Exemple : traceroute google.com.

Structure du système de fichiers

4.3. Gestion des volumes logiques (LVM)

Concepts LVM

  • PV (Physical Volume) : Disques ou partitions physiques.
  • VG (Volume Group) : Regroupement de plusieurs PV.
  • LV (Logical Volume) : Partition logique créée à partir d’un VG.

Commandes principales

  1. Initialisation d’un PV : pvcreate /dev/sda1.
  1. Création d’un VG : vgcreate vg_data /dev/sda1 /dev/sdb1.
  1. Création d’un LV : lvcreate -L 10G -n lv_data vg_data.
  1. Montage d’un LV :
    • Formater : mkfs.ext4 /dev/vg_data/lv_data.
    • Monter : mount /dev/vg_data/lv_data /data.

Redimensionnement

  • Agrandir un LV : lvextend -L +5G /dev/vg_data/lv_data suivi de resize2fs.
  • Réduire un LV : Sauvegarder les données, démonter, puis utiliser lvreduce.

Commandes essentielles

    • Exemple : fsck /dev/sda1.
    • À utiliser avec précaution, surtout sur des systèmes montés.

5. Gestion des utilisateurs et groupes

5.1. Création et gestion des utilisateurs

Commandes essentielles

  1. useradd : Ajouter un utilisateur.
    • Syntaxe : useradd [options] nom_utilisateur.
    • Exemples :
      • Ajouter un utilisateur avec un répertoire personnel : useradd -m john.
      • Spécifier un shell par défaut : useradd -s /bin/bash john.
      • Attribuer un groupe primaire : useradd -g developers john.
  1. usermod : Modifier un utilisateur existant.
    • Syntaxe : usermod [options] nom_utilisateur.
    • Exemples :
      • Changer le répertoire personnel : usermod -d /home/john_new john.
      • Ajouter un utilisateur à un groupe secondaire : usermod -aG sudo john.
      • Verrouiller un compte : usermod -L john.
  1. passwd : Changer ou définir le mot de passe d’un utilisateur.
    • Exemples :
      • Changer le mot de passe : passwd john.
      • Désactiver le mot de passe (compte désactivé) : passwd -l john.
  1. userdel : Supprimer un utilisateur.
    • Syntaxe : userdel [options] nom_utilisateur.
    • Exemples :
      • Supprimer un utilisateur sans ses fichiers : userdel john.
      • Supprimer un utilisateur et son répertoire personnel : userdel -r john.

5.2. Gestion des groupes

Commandes essentielles

  1. groupadd : Ajouter un groupe.
    • Syntaxe : groupadd [options] nom_groupe.
    • Exemple : groupadd developers.
  1. gpasswd : Gestion avancée des groupes.
    • Exemples :
      • Ajouter un utilisateur à un groupe : gpasswd -a john developers.
      • Supprimer un utilisateur d’un groupe : gpasswd -d john developers.
  1. groupmod : Modifier un groupe existant.
    • Changer le nom d’un groupe : groupmod -n newgroup developers.
  1. groupdel : Supprimer un groupe.
    • Exemple : groupdel developers.
  1. Liste des groupes :
    • Afficher tous les groupes : cat /etc/group.
    • Voir les groupes d’un utilisateur : groups john.

5.3. Système de permissions et ACLs

Permissions traditionnelles sous Linux

  1. Structure des permissions :
    • Exemple avec ls -l :
      • rw-r--r-- : Permissions.
        • r (read) : Lecture.
        • w (write) : Écriture.
        • x (execute) : Exécution.
    • Ordre des permissions :
      • Utilisateur propriétaire.
      • Groupe propriétaire.
      • Autres.
  1. Changer les permissions :
    • Commande : chmod.
    • Syntaxe :
      • Mode symbolique : chmod u+rwx,g+r,o-r fichier.txt.
      • Mode numérique : chmod 750 fichier.txt.
      • Exemple d’octal :
        • 7 : Lecture, écriture, exécution (111).
        • 5 : Lecture, exécution (101).
        • 0 : Aucun accès (000).
  1. Changer le propriétaire :
    • Commande : chown.
    • Syntaxe : chown [utilisateur]:[groupe] fichier.txt.
    • Exemples :
      • Changer l’utilisateur propriétaire : chown john fichier.txt.
      • Changer le groupe : chown :users fichier.txt.
      • Changer les deux : chown john:users fichier.txt.
  1. Masque par défaut (umask) :
    • Définit les permissions par défaut.
    • Vérifier : umask.
    • Changer temporairement : umask 022.

ACLs (Access Control Lists)

Les ACLs permettent une gestion plus fine des permissions en ajoutant des règles spécifiques pour des utilisateurs ou groupes particuliers.

  1. Activer les ACLs :
    • Vérifier si le système de fichiers supporte les ACLs : mount | grep acl.
    • Si nécessaire, monter avec l’option : mount -o acl /dev/sda1 /mnt.
  1. Commandes ACLs :
    • Ajouter une règle : setfacl -m u:john:rwx fichier.txt.
      • u : Utilisateur, john.
      • rwx : Lecture, écriture, exécution.
    • Ajouter une règle pour un groupe : setfacl -m g:developers:rw fichier.txt.
    • Supprimer une règle : setfacl -x u:john fichier.txt.
  1. Voir les ACLs :
    • Commande : getfacl fichier.txt.
6. Gestion des services et processus

6.1. Démarrage et arrêt des services

  1. Gestion des services avec systemctl (système systemd) :
    • Démarrer un service : systemctl start service_name.
    • Arrêter un service : systemctl stop service_name.
    • Redémarrer un service : systemctl restart service_name.
    • Recharger la configuration d’un service sans redémarrage complet : systemctl reload service_name.
    • Vérifier l’état d’un service : systemctl status service_name.
  1. Activation/désactivation au démarrage :
    • Activer un service au démarrage : systemctl enable service_name.
    • Désactiver un service au démarrage : systemctl disable service_name.
    • Vérifier si un service est activé : systemctl is-enabled service_name.
  1. Compatibilité avec l’ancien système init.d (commande service) :
    • Démarrer un service : service service_name start.
    • Arrêter un service : service service_name stop.
    • Redémarrer un service : service service_name restart.

6.2. Gestion des niveaux de démarrage et des cibles

  1. Niveaux de démarrage (runlevels) :
    • Les niveaux de démarrage définissent l’état d’un système Linux après le démarrage.
      • 0 : Arrêt du système.
      • 1 : Mode mono-utilisateur (maintenance).
      • 3 : Multi-utilisateurs en mode console.
      • 5 : Multi-utilisateurs avec interface graphique.
      • 6 : Redémarrage.
    • Commande pour afficher le niveau actuel : runlevel.
  1. Cibles (targets) dans systemd :
    • Les cibles remplacent les niveaux de démarrage dans systemd.
      • multi-user.target : Mode multi-utilisateur (équivalent niveau 3).
      • graphical.target : Interface graphique (équivalent niveau 5).
      • rescue.target : Mode mono-utilisateur (équivalent niveau 1).
    • Commandes :
      • Voir la cible actuelle : systemctl get-default.
      • Changer la cible par défaut : systemctl set-default target_name.

8. Gestion des logs et surveillance

8.1. Structure et emplacement des logs système (dmesg, /var/log)

  1. Logs systèmes :
    • Les logs sont essentiels pour diagnostiquer les erreurs, surveiller la sécurité et suivre l’activité du système.
    • Emplacement des logs : La majorité des logs système se trouve dans le répertoire /var/log sur les distributions Linux.
      • /var/log/syslog : Contient les messages généraux du système, utiles pour les diagnostics.
      • /var/log/messages : Contient des informations générales sur le fonctionnement du système.
      • /var/log/auth.log : Consigne les tentatives de connexion, l'authentification des utilisateurs, etc.
      • /var/log/daemon.log : Enregistre les informations liées aux démons (services) du système.
      • /var/log/kern.log : Contient les messages du noyau, souvent utilisés pour le dépannage des erreurs matérielles.
      • /var/log/dmesg : Contient les messages du noyau et du processus d’amorçage (boot).
    • Les fichiers de log peuvent être surveillés et analysés pour détecter des anomalies, des erreurs ou des activités suspectes.
  1. Commandes utiles pour gérer les logs :
    • dmesg : Affiche les messages du noyau et des pilotes. Utile pour voir les erreurs lors du démarrage du système.
      • Exemple : dmesg | grep error pour rechercher des erreurs.
    • journalctl : Outil pour accéder aux logs gérés par systemd. Permet de consulter les journaux système, filtrer par service, etc.
      • Exemple : journalctl -xe pour voir les erreurs récentes.
    • tail -f /var/log/syslog : Permet de suivre en temps réel les nouveaux logs ajoutés.

8.2. Surveillance des ressources système (sar, vmstat, iostat)

  1. Surveillance des ressources système :
    • sar (System Activity Report) : Un outil puissant pour collecter et analyser les performances du système, y compris la CPU, la mémoire, les processus, et le réseau.
      • Exemple : sar -u 1 5 pour surveiller l’utilisation du processeur toutes les 1 seconde, pendant 5 cycles.
      • Les données collectées peuvent être sauvegardées dans des fichiers pour une analyse ultérieure.
    • vmstat (Virtual Memory Statistics) : Fournit un aperçu de l'utilisation de la mémoire, de la gestion des processus, des entrées/sorties, et des caches.
      • Exemple : vmstat 1 5 pour obtenir des statistiques sur la mémoire et les processus chaque seconde, pour 5 cycles.
    • iostat (Input/Output Statistics) : Affiche des informations sur les statistiques des périphériques d’entrée/sortie et des disques.
      • Exemple : iostat -x 1 5 pour surveiller les performances des disques et des périphériques avec des statistiques étendues, toutes les secondes pendant 5 cycles.
  1. Autres outils de surveillance :
    • top et htop : Pour surveiller l'utilisation des ressources CPU, mémoire, et des processus en temps réel.
      • top affiche une liste de processus et l’utilisation des ressources.
      • htop est une version améliorée et plus interactive de top.
    • free : Pour afficher l'utilisation de la mémoire (RAM et swap).
      • Exemple : free -h pour afficher les valeurs de mémoire de manière lisible.

8.3. Mise en place d’outils de monitoring (Nagios, Zabbix, Prometheus)

  1. Nagios :
    • Nagios est un système de surveillance de réseau et d’infrastructure, capable de superviser les hôtes, les services et les ressources systèmes.
    • Il peut alerter en cas de défaillance ou de seuil critique dépassé (p.ex. utilisation CPU élevée, disque plein, etc.).
    • Architecture : Il repose sur une architecture client-serveur, où le serveur Nagios interroge les agents installés sur les hôtes.
    • Configuration de base :
      • Nagios utilise des fichiers de configuration où l'on définit des hôtes et des services à surveiller.
    • Exemple : Surveillance des processus : define service { host_name server1 service_description HTTP check_command check_http }.
  1. Zabbix :
    • Zabbix est un autre outil de surveillance, plus orienté vers l'automatisation et la gestion d'infrastructure complexe.
    • Il permet la collecte de données en temps réel et offre des alertes, une surveillance de la performance des applications, des bases de données, des serveurs et des équipements réseaux.
    • Architecture : Zabbix fonctionne également sur une architecture client-serveur. Il utilise un serveur Zabbix qui collecte les données des hôtes via un agent installé ou SNMP.
    • Exemple : Configurer un hôte dans Zabbix pour surveiller l'utilisation du CPU.
  1. Prometheus :
    • Prometheus est un système de surveillance open-source largement utilisé pour les microservices et les environnements de cloud.
    • Il se distingue par sa capacité à collecter des séries chronologiques de données (metrics) et à exécuter des alertes en fonction de conditions définies.
    • Architecture : Prometheus collecte des métriques via des exporters. Les exporters sont des programmes qui exposent les métriques des services (par exemple, node_exporter pour les données système, mysql_exporter pour MySQL).
    • Exemple : Exposer les métriques du serveur via Prometheus pour la collecte et la surveillance continue.

8.4. Analyse proactive des performances

  1. Collecte régulière des métriques :
    • Il est crucial de collecter régulièrement des données sur la CPU, la mémoire, l’espace disque et les processus.
    • Des outils comme sar et Prometheus permettent d’automatiser cette collecte et de créer des graphiques à partir des séries chronologiques.
  1. Alertes basées sur des seuils :
    • La configuration d'alertes basées sur des seuils de performance (par exemple, 90 % d’utilisation CPU ou mémoire) permet de prévenir de manière proactive les administrateurs des problèmes potentiels.
    • Des outils comme Nagios, Zabbix et Prometheus permettent de définir ces alertes et d'envoyer des notifications par email, SMS ou via des intégrations de chat.
  1. Analyse des tendances :
    • L’analyse des tendances sur plusieurs jours ou semaines peut aider à identifier des problèmes récurrents, comme des hausses d'utilisation de la mémoire qui ne sont pas immédiatement critiques mais qui indiquent des fuites de mémoire ou un sous-dimensionnement des ressources.
    • Outils comme Grafana, associé à Prometheus, permettent de visualiser les tendances et de prévoir des actions correctives à long terme.
  1. Optimisation des performances :
    • Sur la base des métriques collectées, les administrateurs peuvent ajuster la configuration des systèmes, des processus ou des applications.
    • Par exemple, ajuster la configuration de la mémoire virtuelle (swap), optimiser les processus de fond, ou redistribuer les charges de travail entre les serveurs.
9. Sécurité sous Linux

9.1. Bonnes pratiques de sécurité (mise à jour, permissions strictes)

  1. Mise à jour régulière :
    • Maintenir les systèmes à jour est essentiel pour protéger les systèmes contre les vulnérabilités connues.
    • Mises à jour des paquets : Utilisez les gestionnaires de paquets comme apt, yum, ou dnf pour mettre à jour les paquets régulièrement.
      • Exemple (Debian/Ubuntu) : sudo apt update && sudo apt upgrade.
      • Exemple (Red Hat/CentOS) : sudo yum update.
    • Mise à jour du noyau : Un noyau obsolète peut exposer à des attaques. Il est important de vérifier régulièrement la version du noyau et de l'installer s'il est mis à jour.
      • Exemple : uname -r pour connaître la version du noyau en cours.
  1. Permissions strictes :
    • Contrôle des permissions : Assurez-vous que les utilisateurs n'ont accès qu'aux fichiers et ressources nécessaires pour leurs tâches.
      • Utilisez les commandes chmod, chown, et chgrp pour gérer les permissions des fichiers et répertoires.
      • Exemple : chmod 700 /home/utilisateur pour donner un accès exclusif à un utilisateur.
    • Utilisation de la commande umask pour définir des permissions par défaut plus restrictives lors de la création de fichiers.
  1. Principe du moindre privilège :
    • Accorder uniquement les privilèges nécessaires à un utilisateur ou un service. Cela limite les dégâts possibles en cas de compromission.

9.2. Configurer SELinux ou AppArmor

  1. SELinux (Security-Enhanced Linux) :
    • SELinux est une solution de sécurité qui applique un contrôle d'accès obligatoire (MAC). Il impose des politiques de sécurité pour les fichiers, processus et ressources système.
    • Modes SELinux : Il existe trois modes principaux pour SELinux :
      • Enforcing : Les politiques de sécurité sont appliquées.
      • Permissive : Les violations des politiques sont enregistrées, mais pas bloquées.
      • Disabled : SELinux est désactivé.
    • Pour vérifier le mode SELinux :
      • Commande : sestatus.
    • Pour changer le mode :
      • Exemple : sudo setenforce 0 pour passer en mode permissif.
  1. AppArmor :
    • AppArmor est une autre solution de contrôle d'accès obligatoire qui fonctionne de manière similaire à SELinux, mais est plus facile à configurer.
    • Il protège les applications en restreignant les actions qu’elles peuvent effectuer, basées sur des profils définis.
    • Profils : Les profils définissent quelles ressources les applications peuvent accéder. Par défaut, certaines applications comme Firefox ou MySQL sont protégées.
    • Pour vérifier le statut d'AppArmor :
      • Commande : sudo aa-status.

9.3. Gestion des pare-feux

  1. Utilisation de iptables et nftables :
    • iptables : Outil de filtrage de paquets sur Linux. Permet de définir des règles de filtrage pour autoriser ou bloquer le trafic réseau.
      • Exemple : sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT pour autoriser les connexions HTTP sur le port 80.
    • nftables : Nouveau système de filtrage de paquets destiné à remplacer iptables. Il est plus moderne et plus flexible.
      • Exemple : sudo nft add rule ip filter input tcp dport 80 accept.
    • Firewalld : Un outil plus récent, utilisant iptables ou nftables en backend. Il offre une interface plus simple pour gérer les règles de pare-feu.
      • Exemple : sudo firewall-cmd --zone=public --add-service=http.
  1. Principes de base du pare-feu :
    • Bloquer tout par défaut, puis autoriser uniquement les services nécessaires.
    • Créer des règles en fonction des zones de sécurité (par exemple, public, dmz).
    • Exemple de configuration :
      • Bloquer tout le trafic entrant par défaut : sudo iptables -P INPUT DROP.
      • Autoriser le SSH : sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT.
  1. Surveillance du pare-feu :
    • iptables -L : Liste les règles actuellement appliquées par iptables.
    • nft list ruleset : Affiche les règles de nftables.

9.4. Mise en place de VPN (OpenVPN, WireGuard)

  1. OpenVPN :
    • OpenVPN est une solution VPN open-source, très utilisée pour sécuriser les connexions réseau sur Internet.

    • Exemple pour installer OpenVPN :

    • Exemple pour démarrer un serveur OpenVPN :
      • sudo systemctl start openvpn@server.
  1. WireGuard :
    • WireGuard est une technologie VPN plus récente, offrant des performances supérieures et une configuration simplifiée par rapport à OpenVPN.
    • Il utilise des clés publiques/privées pour établir des connexions sécurisées.
    • Installation de WireGuard :
      • sudo apt install wireguard (Debian/Ubuntu).

  1. Systèmes de détection d'intrusion (IDS) :
    • Les IDS surveillent le trafic réseau et les actions sur le système pour détecter des comportements anormaux, souvent associés à des attaques.

  1. Systèmes de prévention des intrusions (IPS) :
    • Un IPS va au-delà de la simple détection et peut automatiquement bloquer le trafic malveillant en fonction des politiques définies.
    • Suricata est un exemple d’IPS open-source utilisé pour l’analyse du trafic réseau.
      • Exemple de configuration :
        • sudo apt install suricata.
  1. Logs et alertes :
    • La gestion des logs d'IDS/IPS est essentielle pour analyser et réagir aux menaces.
    • Utilisation de syslog ou journalctl pour centraliser et analyser les alertes générées par les systèmes IDS/IPS.
    • En cas d'attaque détectée, le système doit alerter l'administrateur et potentiellement bloquer l'IP ou l’activité suspecte en utilisant des règles de pare-feu.

    • Ansible est agentless, c'est-à-dire qu'il ne nécessite aucun logiciel client installé sur les machines de gestion.
    • Puppet nécessite un agent installé sur chaque machine cible, mais il est plus robuste pour la gestion à grande échelle.
    • Ansible est souvent préféré pour sa simplicité et sa rapidité, tandis que Puppet est plus adapté aux environnements très complexes nécessitant des règles détaillées de gestion de configuration.

    • LPIC-1 : Compétences de base en ligne de commande, gestion des fichiers et processus.
    • LPIC-2 : Administration système avancée, réseaux, et virtualisation.
    • LPIC-3 : Expertise en virtualisation, sécurité et gestion d'environnements mixtes.
    • Préparation :
      • Étudier les concepts multi-distributions (Debian, Red Hat, SuSE).
      • Accéder aux guides officiels et exercices interactifs.
  1. CompTIA Linux+ :
    • Certification indépendante des distributions, idéale pour débutants/intermédiaires.
    • Couvre les bases de l'administration, la gestion des processus, la sécurité et les réseaux.
    • Préparation :
      • Utiliser les ressources officielles CompTIA.
