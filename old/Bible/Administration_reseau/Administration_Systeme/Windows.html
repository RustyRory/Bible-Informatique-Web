Windows

1. Introduction à l'administration Windows

1.1. Comprendre les rôles et responsabilités d’un administrateur Windows

Un administrateur Windows est responsable de la gestion, du maintien et de l'optimisation des environnements Windows Server. Ses principales tâches incluent :

  • Gestion des utilisateurs et des groupes : Création, modification et suppression d'utilisateurs, gestion des permissions et authentifications.
  • Supervision des systèmes : Surveillance des performances, mise à jour des correctifs, et dépannage des pannes.
  • Sécurité : Implémentation des politiques de sécurité, gestion des pare-feu, détection et prévention des intrusions.
  • Gestion des réseaux : Configuration des interfaces réseau, des serveurs DHCP, DNS, et VPN.
  • Automatisation : Utilisation d’outils comme PowerShell pour automatiser les tâches administratives.

Un bon administrateur doit également anticiper les problèmes en surveillant l’infrastructure et en prenant des mesures proactives.

1.2. Versions et éditions de Windows Server

Microsoft propose plusieurs éditions de Windows Server, adaptées à différents besoins :

  • Standard Edition : Destinée aux entreprises nécessitant des fonctionnalités essentielles de serveur, avec une limite sur les machines virtuelles (2 VM maximum).
  • Datacenter Edition : Adaptée aux environnements à forte densité de virtualisation, offrant des fonctionnalités avancées comme Storage Spaces Direct et Shielded VMs.
  • Essentials Edition : Conçue pour les petites entreprises (moins de 25 utilisateurs ou 50 appareils), avec une gestion simplifiée et des fonctionnalités limitées.

Versions courantes :

  • Windows Server 2016 : Ajout de Nano Server et Windows Containers.
  • Windows Server 2019 : Intégration avec Azure, améliorations de la sécurité.
  • Windows Server 2022 : Optimisations pour le Cloud hybride et une meilleure sécurité avec Secured-core Server.

1.3. Interface utilisateur Windows Server : GUI vs Core (Server Core)

Windows Server est disponible en deux modes d'installation principaux :

  1. Interface graphique complète (GUI) :
    • Idéal pour les administrateurs débutants ou les environnements nécessitant des interfaces visuelles.
    • Permet l’utilisation des outils graphiques pour la configuration, mais consomme plus de ressources système.
  1. Server Core :
    • Une version minimaliste sans interface graphique, accessible uniquement via la ligne de commande ou à distance.
    • Avantages : meilleure sécurité, consommation réduite de ressources, et surface d'attaque moindre.
    • Utilisée dans les datacenters et pour les serveurs nécessitant une haute disponibilité.

1.4. Installation et configuration initiale de Windows Server

La mise en place d’un serveur Windows nécessite plusieurs étapes importantes :

  1. Préparation avant installation :
    • Identifier les besoins en termes de stockage, de mémoire et de processeur.
    • Choisir le type d’installation : GUI ou Server Core.
    • Préparer les partitions disque pour le système d'exploitation.
  1. Processus d'installation :
    • Lancer le support d’installation (DVD, USB ou réseau).
    • Suivre les étapes : sélectionner l’édition, configurer les partitions et installer.
    • Appliquer les mises à jour post-installation via Windows Update.
  1. Configuration initiale :
    • Définir un nom d’hôte : Identifier le serveur sur le réseau.
    • Configurer les paramètres réseau : Adressage IP, passerelle et serveurs DNS.
    • Créer des utilisateurs initiaux : Comptes administratifs et comptes de service.
    • Configurer les locales : Langue, fuseau horaire, et format des dates.

La configuration initiale garantit que le serveur est prêt pour les rôles ou fonctionnalités qui lui seront assignés ultérieurement (ex. AD, DNS, Hyper-V).

2.1. Gestion des utilisateurs et des groupes locaux

  • Utilisateurs locaux : Comptes créés et gérés sur un seul serveur, utiles pour les petites infrastructures ou des configurations spécifiques.
    • Commandes courantes :
      • net user <nom_utilisateur> : Créer ou modifier un utilisateur local.
      • lusrmgr.msc : Interface graphique pour gérer les utilisateurs et groupes locaux.

  • Groupes locaux : Rassemblent des utilisateurs avec des permissions similaires.
    • Exemples : Administrateurs (permissions maximales), Utilisateurs (permissions standard).

3. Active Directory (AD) et gestion des domaines

3.1. Concepts fondamentaux d’Active Directory (forêts, arbres, domaines)

Active Directory (AD) est une infrastructure de gestion centralisée des ressources et des utilisateurs dans un environnement Windows. Voici les concepts clés :

  • Forêt (Forest) :
    • L'entité la plus élevée dans AD.
    • Représente une instance unique d'Active Directory contenant un ou plusieurs arbres.
    • Toutes les entités de la forêt partagent un schéma commun (définition des types d’objets, propriétés).
  • Arbre (Tree) :
    • Un ensemble de domaines liés par des relations hiérarchiques.
    • Les domaines d'un arbre partagent un espace de noms contigu. Par exemple :
      • Domaine parent : exemple.com.
      • Domaine enfant : marketing.exemple.com.
  • Domaine :
    • Un conteneur administratif regroupant des objets AD tels que des utilisateurs, des groupes et des ressources.
    • Il constitue une unité de sécurité : chaque domaine a son propre ensemble de stratégies et d’autorisations.
  • Contrôleur de domaine (Domain Controller - DC) :
    • Serveur exécutant Active Directory Domain Services (AD DS).
    • Gère les connexions des utilisateurs, l’authentification et la réplication des données entre DCs.

3.2. Configuration et gestion des contrôleurs de domaine

Un contrôleur de domaine est essentiel pour gérer les ressources AD et assurer la disponibilité du réseau.

  • Installation d’AD DS sur un serveur :
    • Utiliser le gestionnaire de serveur pour ajouter le rôle Active Directory Domain Services.
    • Promouvoir le serveur en tant que contrôleur de domaine.
    • Configurer un nouveau domaine ou joindre un domaine existant.
  • Répartition des rôles FSMO (Flexible Single Master Operations) :
    • Les rôles FSMO sont critiques pour le fonctionnement d’AD :
      • Maître de schéma : Uniquement dans la forêt, gère le schéma AD.
      • Maître de noms de domaine : Gestion des espaces de noms uniques dans la forêt.
      • RID Master : Génère les pools d'identificateurs pour les objets AD.
      • PDC Emulator : Synchronise l’heure et gère les mots de passe dans le domaine.
      • Infrastructure Master : Gère les références d’objets entre domaines.
  • Sauvegarde et restauration d’Active Directory :
    • Utiliser l’outil Windows Server Backup ou wbadmin.
    • Mode de restauration des services d’annuaire (Directory Services Restore Mode - DSRM) pour restaurer AD.

3.3. Gestion des stratégies de groupe (Group Policy Objects - GPO)

Les GPOs permettent d’appliquer des configurations système et des restrictions aux utilisateurs ou aux ordinateurs dans un domaine.

  • Types de GPOs :
    • Local : Appliqué à un ordinateur spécifique, même hors domaine.
    • Domaine : GPOs définis dans AD, appliqués aux utilisateurs et ordinateurs d’un domaine.
    • GPO Liés : Appliqués à des OU spécifiques ou des groupes de domaines.
  • Outils de gestion des GPOs :
    • Group Policy Management Console (GPMC) : Gestion graphique des GPOs.
    • Commandes PowerShell :
      • Créer une nouvelle GPO : 
        New-GPO -Name "NomDeLaGPO"
      • Appliquer une GPO à une OU : 
        New-GPLink -Name "NomDeLaGPO" -Target "OU=Test,DC=domaine,DC=com"
  • Exemples d’utilisation des GPOs :
    • Appliquer des politiques de mot de passe.
    • Configurer des restrictions logicielles.
    • Déployer des scripts de démarrage/arrêt.

3.4. Configuration des relations d'approbation entre domaines et forêts

Les relations d'approbation permettent aux utilisateurs de différents domaines ou forêts de partager des ressources tout en conservant leurs identités respectives.

  • Types de relations d’approbation :
    • Unidirectionnelle : Les utilisateurs d’un domaine peuvent accéder aux ressources d’un autre, mais pas l’inverse.
    • Bidirectionnelle : Accès mutuel entre deux domaines.
    • Approche transitive : Les approbations se propagent à travers une hiérarchie d'espaces de noms.
  • Configuration des relations d’approbation :
    • Accéder à Domain and Trusts dans l’interface graphique.
    • Ajouter une nouvelle relation en spécifiant les domaines concernés.
    • Commande PowerShell : 
      New-AdDomainTrust -Name "domaineA.com" -TargetName "domaineB.com" -TrustType External -Direction Bidirectional
  • Scénarios courants :
    • Approche entre deux forêts distinctes.
    • Consolidation des domaines lors d’une fusion d’entreprises.
4. Gestion des rôles et des fonctionnalités de Windows Server

4.1. Serveur DHCP : Configuration et gestion des baux IP

Le Dynamic Host Configuration Protocol (DHCP) automatise l'attribution des adresses IP dans un réseau. Il réduit les erreurs humaines et facilite l'administration.

Principales fonctions du serveur DHCP :

  • Attribution dynamique des adresses IP : Les clients reçoivent automatiquement des adresses IP valides avec des informations réseau (passerelle, DNS).
  • Gestion des baux (leases) : Durée pendant laquelle une IP est réservée pour un appareil. Une fois expirée, l'IP peut être réattribuée.
  • Réservations : Associer une adresse IP spécifique à une adresse MAC pour garantir une IP fixe.
  • Options DHCP : Définir des paramètres supplémentaires (DNS, passerelle, nom de domaine).

Étapes pour configurer un serveur DHCP :

  1. Ajouter le rôle DHCP :
    • Depuis le Gestionnaire de serveur, ajouter le rôle DHCP.
  1. Configurer les plages d’adresses (scopes) :
    • Définir la plage IP (par ex. 192.168.1.100 à 192.168.1.200).
    • Configurer les options DHCP comme les serveurs DNS et la passerelle.
  1. Activer le scope : Le rendre disponible pour les clients.
  1. Supervision :
    • Utiliser la console DHCP pour surveiller les baux actifs et gérer les réservations.

4.2. Serveur DNS : Zones, enregistrements, et résolutions de noms

Le Domain Name System (DNS) traduit les noms de domaine en adresses IP, permettant aux utilisateurs d'accéder facilement aux ressources réseau.

Éléments essentiels du serveur DNS :

  • Zones DNS : Contiennent les enregistrements DNS pour une portion de l’espace de noms.
    • Zone principale (Primary Zone) : Zone écrivable contenant les données principales.
    • Zone secondaire (Secondary Zone) : Copie en lecture seule d’une zone principale pour la redondance.
    • Zone de recherche inversée (Reverse Lookup Zone) : Permet de résoudre une adresse IP en nom de domaine.
  • Enregistrements DNS :
    • A (Address) : Associe un nom de domaine à une adresse IPv4.
    • AAAA : Associe un nom de domaine à une adresse IPv6.
    • CNAME (Alias) : Redirige un nom de domaine vers un autre.
    • MX (Mail Exchange) : Indique les serveurs de messagerie.
    • PTR : Utilisé dans les zones de recherche inversée.
  • Résolution de noms :
    • Récursive : Le serveur DNS interroge d'autres serveurs jusqu'à obtenir une réponse.
    • Itérative : Le client interroge plusieurs serveurs DNS lui-même.

Configuration d'un serveur DNS :

  1. Ajouter le rôle DNS via le Gestionnaire de serveur.
  1. Créer une zone (primaire ou secondaire) et y ajouter des enregistrements.
  1. Configurer les paramètres de résolution :
    • Serveurs DNS pour la récursivité ou les transferts conditionnels.
  1. Superviser les requêtes DNS via la console ou PowerShell.

4.3. Serveur de fichiers et stockage : Quotas et DFS (Distributed File System)

Quotas de disque :

Permettent de limiter l'espace utilisé par les utilisateurs ou groupes sur un volume.

  • Fonctionnalités principales :
    • Quotas utilisateur : Définit un espace maximum utilisable pour chaque utilisateur.
    • Seuils et alertes : Envoi d'alertes quand une limite est presque atteinte.
  • Configuration des quotas :
    1. Activer les quotas sur un volume via les propriétés du disque.
    1. Définir les limites et les seuils.

DFS (Distributed File System) :

DFS facilite la gestion des données dans un réseau en permettant un accès centralisé à des dossiers distribués sur plusieurs serveurs.

  • Types de DFS :
    • DFS Namespaces : Crée une vue unifiée de fichiers dispersés sur plusieurs serveurs.
    • DFS Replication : Réplique des données entre plusieurs serveurs pour garantir la disponibilité.
  • Avantages :
    • Haute disponibilité des données.
    • Réduction des points de défaillance.
  • Configuration de DFS :
    1. Ajouter le rôle DFS via le Gestionnaire de serveur.
    1. Créer un Namespace DFS.
    1. Ajouter des cibles de dossier et configurer la réplication.

4.4. Rôles d’impression : Gestion des imprimantes réseau et des spouleurs

Le rôle d'impression centralise la gestion des imprimantes réseau.

Fonctionnalités du serveur d'impression :

  • Partage d’imprimantes : Permet aux utilisateurs de se connecter à une imprimante via le serveur.
  • Gestion des files d’attente (spoolers) : Surveille les tâches d’impression et résout les blocages.
  • Déploiement par GPO : Distribue les imprimantes aux utilisateurs ou groupes via les Stratégies de Groupe.

Configuration d’un serveur d’impression :

  1. Ajouter le rôle d’impression via le Gestionnaire de serveur.
  1. Installer et partager une imprimante :
    • Ajouter les pilotes nécessaires.
    • Partager l’imprimante sur le réseau.
  1. Superviser les files d’attente via la console de gestion d'impression.
  1. Optimisation :
    • Configurer des permissions pour limiter l'accès.
    • Activer la journalisation pour surveiller l'utilisation.
5. Virtualisation avec Hyper-V

5.1. Concepts de base de la virtualisation avec Hyper-V

Définition et rôle d’Hyper-V :

Hyper-V est un hyperviseur de type 1 intégré aux éditions Windows Server (et certaines éditions de Windows 10/11 Pro et Enterprise). Il permet de créer et de gérer des machines virtuelles (VM) en optimisant l'utilisation des ressources physiques.

Avantages de la virtualisation avec Hyper-V :

  • Efficacité des ressources : Partage des ressources physiques (CPU, mémoire, disque) entre plusieurs VM.
  • Isolation : Chaque VM fonctionne indépendamment, ce qui réduit les risques liés à un problème sur une autre VM.
  • Flexibilité : Possibilité de tester, développer ou héberger plusieurs environnements sans matériel supplémentaire.
  • Sauvegarde et migration : Facilité de sauvegarde des VM et mobilité entre serveurs physiques grâce à la migration à chaud (Live Migration).

Composants clés d’Hyper-V :

  • Hyperviseur : Gère l'accès des VM aux ressources matérielles.
  • Gestionnaire Hyper-V : Interface graphique pour configurer et administrer les VM.
  • Réseaux virtuels : Permettent aux VM de communiquer entre elles, avec l’hôte et avec l’extérieur.

5.2. Configuration des machines virtuelles (VM) et des réseaux virtuels

Création de machines virtuelles (VM) :

  1. Installation du rôle Hyper-V :
    • Utiliser le Gestionnaire de serveur pour ajouter le rôle Hyper-V.
  1. Création d’une VM via le Gestionnaire Hyper-V :
    • Spécifier le nom et l’emplacement de la VM.
    • Configurer les ressources de la VM :
      • Mémoire (statique ou dynamique).
      • Nombre de cœurs CPU.
      • Taille et emplacement du disque dur virtuel (VHD/VHDX).
    • Choisir un système d'exploitation invité (ISO, CD/DVD virtuel).
  1. Démarrage et configuration :
    • Démarrer la VM et installer l'OS invité.
    • Installer les Hyper-V Integration Services pour améliorer les performances et la compatibilité.

Réseaux virtuels dans Hyper-V :

Hyper-V utilise des commutateurs virtuels pour gérer la connectivité des VM :

  • Commutateur externe : Connecte les VM au réseau physique.
  • Commutateur interne : Permet aux VM et à l’hôte de communiquer entre eux sans accès à l’extérieur.
  • Commutateur privé : Limite la communication entre les VM sans interaction avec l’hôte ou le réseau physique.

5.3. Sauvegarde, exportation et restauration des VM

Sauvegarde des VM :

Hyper-V prend en charge des sauvegardes à chaud (sans interruption des VM), utilisant la technologie Volume Shadow Copy Service (VSS).

  • Outils intégrés :
    • Windows Server Backup.
    • Fonctionnalités de sauvegarde de solutions tierces compatibles Hyper-V.
  • Étapes principales :
    1. Configurer un emplacement pour stocker les sauvegardes (local ou distant).
    1. Planifier des sauvegardes régulières.
    1. Vérifier les journaux pour s’assurer de la réussite des sauvegardes.

Exportation et importation des VM :

  • Exportation : Crée un fichier contenant la VM complète (état, configuration, et disques virtuels).
    • Utiliser l’option Exporter dans le Gestionnaire Hyper-V.
  • Importation : Restaure une VM exportée.
    • Choisir le mode d'importation :
      • Enregistrement : Réutilise l’ID de la VM.
      • Copie : Crée un nouvel ID, utile pour dupliquer une VM.
      • Restauration : Restaure la VM dans son état initial.

Restauration des VM :

  • Utiliser une sauvegarde complète ou les snapshots (points de contrôle) pour revenir à un état antérieur.
  • Commande PowerShell pour restaurer : 
    Resize-VHD -Path "CheminVersLeDisque.vhdx" -SizeBytes <NouvelleTaille>
6. Réseaux sous Windows Server

6.1. Configuration des interfaces réseau : IPv4, IPv6, routage

Configuration des interfaces réseau IPv4 et IPv6 :

Windows Server permet la configuration des interfaces réseau en utilisant des protocoles IPv4 et IPv6 pour assurer une communication efficace sur un réseau local ou à travers Internet.

IPv4 :

  • Attribution d'adresses IP : Peut être configuré statiquement (adresses fixes) ou dynamiquement (via DHCP).
  • Configuration manuelle d’une adresse IPv4 :
    1. Ouvrir Panneau de configuration > Réseau et Internet > Connexions réseau.
    1. Cliquer avec le bouton droit sur l'interface réseau, puis Propriétés.
    1. Sélectionner Protocole Internet version 4 (TCP/IPv4) et configurer les paramètres :
      • Adresse IP : Spécifier l'adresse IP statique.
      • Masque de sous-réseau : Définit la taille du réseau.
      • Passerelle par défaut : Adresse du routeur pour accéder à d'autres réseaux.
      • Serveur DNS : Serveur pour résoudre les noms de domaine.

IPv6 :

  • Le processus est similaire à celui de IPv4, mais pour IPv6, l’adresse IP est plus complexe (ex. fe80::abcd:1234:5678:abcd).
  • IPv6 permet une plus grande quantité d'adresses, ce qui est essentiel dans les réseaux modernes.

Détails supplémentaires :

  • DHCP (Dynamic Host Configuration Protocol) : Utilisé pour attribuer automatiquement des adresses IP aux appareils du réseau.
  • Saisie manuelle des adresses IPv4/IPv6 ou configuration via DHCP est souvent choisie selon les besoins spécifiques du réseau.

Routage :

Le routage permet de diriger le trafic réseau entre différents sous-réseaux ou réseaux. Sur un serveur Windows, cela peut être configuré via le rôle Services de routage et d'accès à distance (RRAS).

  • Activation du routage :
    1. Dans le Gestionnaire de serveur, ajouter le rôle Services de routage et d'accès à distance.
    1. Configurer Routage IP et NAT (Network Address Translation) si nécessaire pour permettre la communication entre les sous-réseaux.
    1. Ajouter des routes statiques ou utiliser le Routage dynamique (RIP, OSPF).

6.2. Gestion des pare-feux Windows Defender

Le pare-feu Windows Defender est un composant essentiel pour sécuriser le serveur et les réseaux internes en bloquant ou autorisant des connexions réseau basées sur des règles définies.

Configuration du pare-feu :

  1. Accès au pare-feu :
    • Ouvrir Panneau de configuration > Système et sécurité > Pare-feu Windows Defender.
    • Ou via Windows Admin Center pour une gestion plus centralisée.
  1. Création de règles de trafic entrant et sortant :
    • Une règle peut être configurée pour autoriser ou bloquer des ports spécifiques ou des adresses IP.
    • Exemple : Autoriser le trafic HTTP (port 80) :
      1. Accédez à Règles de trafic entrant.
      1. Cliquez sur Nouvelle règle, sélectionnez Port, puis entrez le port (80).
      1. Choisissez Autoriser la connexion.
  1. Règles pour les applications : Permet de spécifier quelles applications peuvent recevoir des connexions réseau.

Exemples de configuration avancée :

  • Autoriser le trafic RDP (port 3389) pour la connexion à distance.
  • Configurer des règles pour VPN, FTP, ou SQL Server.

Sécurisation du pare-feu :

  • Profile du réseau : Le pare-feu peut avoir des paramètres différents selon le profil (Public, Privé ou Domaine).
  • Activation des alertes : Configurer des alertes pour les tentatives de connexions non autorisées.
  • Filtrage des connexions sortantes : Bloquer les connexions sortantes non nécessaires.

6.3. Sécurisation des connexions distantes avec RDP (Remote Desktop Protocol)

Le RDP est un protocole largement utilisé pour accéder à distance à un serveur Windows. Pour sécuriser RDP, il est essentiel de configurer le pare-feu, les stratégies d'authentification et de surveillance des connexions.

Activer le RDP :

  1. Ouvrir Système dans le Panneau de configuration.
  1. Sélectionner Paramètres d’accès à distance.
  1. Activer l'option Autoriser les connexions à distance à cet ordinateur.

Sécurisation de RDP :

  • Utilisation de TLS (Transport Layer Security) pour sécuriser les sessions RDP et chiffrer les données échangées.
  • Authentification forte : Utiliser des certificats SSL/TLS pour s'assurer que les connexions RDP proviennent de sources sûres.
  • Restriction des connexions :
    • Limiter l'accès à certaines adresses IP.
    • Configurer le pare-feu pour permettre le trafic RDP uniquement sur des réseaux spécifiques (ex. VPN).
    • Désactiver les connexions RDP pour les utilisateurs non autorisés.

Gestion des connexions RDP :

  • Limitation du nombre de sessions : Configurer une limite de nombre de connexions simultanées par utilisateur.
  • Stratégies de groupe : Utiliser des GPO pour restreindre l’accès aux utilisateurs ou groupes spécifiques.

6.4. Mise en place d’un accès VPN (DirectAccess, Always On VPN)

Les VPN (Virtual Private Network) permettent aux utilisateurs distants de se connecter au réseau de l'entreprise de manière sécurisée, tout en simulant une connexion locale. Windows Server propose plusieurs solutions pour configurer un accès VPN.

DirectAccess :

DirectAccess est une technologie Microsoft qui permet aux utilisateurs distants de se connecter de manière transparente à un réseau privé sans avoir besoin d'une connexion VPN traditionnelle. Il repose sur IPv6 et permet une connexion toujours active pour les utilisateurs, avec une gestion automatique des connexions.

  • Configuration :
    1. Ajouter le rôle DirectAccess et VPN via Gestionnaire de serveur.
    1. Configurer la connexion DirectAccess pour permettre aux utilisateurs distants d’accéder aux ressources internes.

Always On VPN :

Always On VPN est une solution moderne et plus sécurisée que DirectAccess. Elle permet aux utilisateurs distants de se connecter automatiquement à l'entreprise chaque fois qu'ils se connectent à Internet, offrant une expérience transparente et sécurisée.

  • Configuration :
    1. Ajouter le rôle Always On VPN via Services d’accès à distance.
    1. Créer des profils VPN pour les utilisateurs via Server Manager.
    1. Configurer les certificats et les stratégies de groupe (GPO) pour gérer les connexions VPN.

Protocoles VPN dans Windows Server :

  • PPTP : Très rapide mais obsolète et moins sécurisé.
  • L2TP/IPsec : Plus sécurisé que PPTP.
  • SSTP : VPN sécurisé avec SSL, souvent utilisé pour contourner les pare-feu stricts.
  • IKEv2/IPsec : Une option moderne et sécurisée.
7. Sécurité et gestion des identités

7.1. Concepts de base de la sécurité sous Windows Server

Principes de base de la sécurité sous Windows Server :

La sécurité sous Windows Server repose sur plusieurs principes fondamentaux pour protéger les systèmes et les données :

  1. Contrôle d'accès :
    • Les utilisateurs et les groupes ont des autorisations qui déterminent l'accès aux ressources. Les permissions peuvent être configurées sur des fichiers, des dossiers et des objets dans Active Directory.
    • Contrôle d'accès basé sur des rôles (RBAC) : Attribuer des rôles spécifiques aux utilisateurs pour limiter leur accès aux ressources nécessaires.
  1. Stratégies de sécurité :
    • Stratégies de groupe (GPO) : Permet de définir des politiques de sécurité centralisées pour l'ensemble du domaine. Elles permettent de configurer des paramètres de sécurité tels que les mots de passe, l'audit des connexions, et les paramètres du pare-feu.
    • Contrôle des comptes utilisateurs (UAC) : Limite les droits des utilisateurs pour éviter l'exécution de programmes malveillants, même si ces derniers sont exécutés avec des privilèges d'administrateur.
  1. Sécurisation des communications :
    • Chiffrement des données en transit et au repos (par exemple, via SSL/TLS, BitLocker, etc.) pour protéger les informations sensibles.
    • Protocoles sécurisés comme SSH pour l'administration distante et RDP pour les connexions à distance.
  1. Audit et surveillance :
    • Mise en œuvre de journaux d’audit pour surveiller les activités et détecter les comportements anormaux. Ces journaux incluent les connexions réussies ou échouées, les changements de groupe, et les tentatives d'accès non autorisées.
  1. Mises à jour et correctifs :
    • Il est essentiel de maintenir le système à jour avec les dernières mises à jour de sécurité et de déployer rapidement des correctifs critiques pour limiter l’exposition aux vulnérabilités.

7.2. Gestion des certificats avec AD Certificate Services (AD CS)

Active Directory Certificate Services (AD CS) est un rôle de Windows Server qui fournit une infrastructure de gestion des certificats pour sécuriser la communication et l'authentification dans un réseau.

Principes de fonctionnement de AD CS :

  • Certificat numérique : Un certificat est un document numérique utilisé pour vérifier l'identité d'un utilisateur, d'un appareil, ou d'un serveur, ainsi que pour chiffrer des données.
  • Autorité de certification (CA) : Une CA est un serveur qui émet et gère les certificats. AD CS permet d'implémenter des CA au sein de l'entreprise.
  • Types de certificats : AD CS peut émettre plusieurs types de certificats :
    • Certificats de serveur (pour l’authentification SSL/TLS).
    • Certificats d'utilisateur (pour l'authentification et le chiffrement).
    • Certificats pour la signature des e-mails.

Mise en place d’AD CS :

  1. Ajouter le rôle Services de certificat Active Directory via Gestionnaire de serveur.
  1. Configurer une Autorité de certification (CA) racine ou secondaire selon l'architecture de sécurité.
  1. Définir les politiques de certificat pour déterminer la durée de validité, la méthode d’émission, et la révocation des certificats.
  1. Distribuer et gérer les certificats à travers des Groupes de stratégie de certificat (GPO).

Avantages de AD CS :

  • Automatisation de l'émission et de la gestion des certificats.
  • Amélioration de la sécurité via l'authentification mutuelle et le chiffrement des communications.
  • Gestion centralisée des certificats dans l'environnement Active Directory.

7.3. Mise en place de l’authentification multi-facteurs (MFA)

L'authentification multi-facteurs (MFA) est un mécanisme de sécurité qui exige plus d’une méthode d’authentification pour accéder à un système, réduisant ainsi les risques d’accès non autorisé.

Principes de MFA :

MFA combine plusieurs des trois facteurs suivants :

  • Quelque chose que vous savez (par exemple, un mot de passe ou un PIN).
  • Quelque chose que vous avez (par exemple, un téléphone mobile, une carte à puce, ou un jeton de sécurité).
  • Quelque chose que vous êtes (par exemple, une empreinte digitale ou une reconnaissance faciale).

Mise en place de MFA sous Windows Server :

  1. Windows Server et Azure AD :
    • L'intégration de Azure Active Directory avec MFA permet d'étendre les capacités de sécurité aux environnements cloud. Vous pouvez appliquer MFA via Microsoft Authenticator, des SMS ou des appels vocaux.
    • Paramétrer Azure MFA dans le Azure AD Portal.
  1. Active Directory et MFA :
    • Utiliser des solutions tiers comme Microsoft Advanced Threat Analytics (ATA) ou Windows Hello for Business pour implémenter MFA sur les comptes utilisateurs locaux.
    • Les politiques MFA peuvent être appliquées via des stratégies de groupe ou des GPO pour exiger une validation supplémentaire lors des connexions aux ressources sensibles.
  1. Exemples de scénarios MFA :
    • Connexion à des services distants via RDP.
    • Accès à des applications critiques ou à des ressources via un réseau privé (VPN).
    • Chiffrement des e-mails en exigeant un deuxième facteur avant de déchiffrer les informations.

7.4. Utilisation de Windows Defender et Microsoft Security Baseline

Windows Defender :

Windows Defender Antivirus est un logiciel de sécurité intégré qui offre une protection contre les virus, les malwares, et les attaques en temps réel. Il fait partie de la suite de sécurité Windows Server.

Principales fonctionnalités :

  1. Protection en temps réel contre les malwares et les logiciels indésirables.
  1. Analyse de sécurité pour rechercher les menaces potentielles et les vulnérabilités.
  1. Protection contre les ransomwares en utilisant des outils comme Controlled Folder Access.
  1. Gestion centralisée des alertes de sécurité via Windows Defender Security Center ou System Center Configuration Manager (SCCM).

Microsoft Security Baseline :

Les Microsoft Security Baselines sont des ensembles de recommandations de sécurité basées sur les meilleures pratiques de l'industrie, permettant de configurer et de maintenir un niveau de sécurité optimal pour les serveurs Windows.

Composants des baselines de sécurité :

  1. Stratégies de groupe (GPO) : Configurations de sécurité prédéfinies, telles que les paramètres de mot de passe, les comptes à privilèges élevés, la gestion des mises à jour, et les paramètres de sécurité réseau.
  1. Windows Defender : Paramètres de protection comme les exclusions, les contrôles d'accès et la surveillance des comportements suspects.

Application des Microsoft Security Baselines :

8. Gestion du stockage

8.1. Configuration des disques et volumes : disques dynamiques, RAID logiciel

Disques de base vs Disques dynamiques :

  • Disques de base : Ce sont les types de disques traditionnels, utilisés dans les configurations simples comme les disques simples ou les partitions de base. Ils sont limités à des configurations de volumes simples.
  • Disques dynamiques : Offrent une plus grande flexibilité. Les disques dynamiques permettent de créer des volumes complexes, comme les volumes répartis, les volumes en miroir, et les volumes RAID logiciel.

Types de volumes sur disques dynamiques :

  1. Volume simple : Un volume constitué d'un seul disque. C’est une configuration de base.
  1. Volume fractionné : Un volume qui s'étend sur plusieurs disques. Il améliore la capacité de stockage en utilisant l'espace sur plusieurs disques.
  1. Volume miroir (RAID 1) : Réplique les données d'un disque sur un autre pour garantir la redondance des données et la tolérance aux pannes.
  1. Volume en parité (RAID 5) : Répartit les données et la parité entre trois disques ou plus. Cela permet d’optimiser l’espace de stockage tout en assurant la redondance des données.
  1. Volume RAID 10 (RAID 1+0) : Combinaison de volumes en miroir et fractionnés. Offre une haute disponibilité et des performances élevées, mais nécessite un minimum de quatre disques.

RAID logiciel sous Windows Server :

  • RAID 1 (Miroir) : Les données sont répliquées sur deux disques pour garantir la tolérance aux pannes.
  • RAID 5 (Parité répartie) : Nécessite au moins trois disques et utilise une parité pour garantir la redondance tout en optimisant l'espace disque.
  • RAID 10 (RAID 1+0) : Combine les avantages du RAID 1 (redondance) et du RAID 0 (performance), mais nécessite un minimum de quatre disques.

Gestion des volumes :

  • Utiliser l’outil Gestion des disques pour configurer les disques dynamiques et gérer les volumes RAID logiciel.
  • La configuration des volumes RAID logiciel peut se faire via PowerShell ou l'interface graphique de gestion des disques dans le Gestionnaire de serveur.

8.2. Introduction aux systèmes de fichiers (NTFS, ReFS)

NTFS (New Technology File System) :

  • NTFS est le système de fichiers le plus utilisé dans les systèmes Windows Server. Il prend en charge les grandes tailles de volume et offre des fonctionnalités avancées comme la sécurité, le chiffrement et la gestion des quotas.
  • Avantages de NTFS :
    • Sécurisation des fichiers via les listes de contrôle d'accès (ACL).
    • Chiffrement des fichiers avec EFS (Encrypting File System).
    • Compression de fichiers pour économiser de l'espace.
    • Support de la récupération après une panne grâce à des journaux de transactions.

ReFS (Resilient File System) :

  • ReFS est un système de fichiers introduit dans Windows Server 2012, conçu pour offrir une meilleure résilience, surtout pour les environnements de stockage de données critiques.
  • Avantages de ReFS :
    • Résilience : ReFS peut détecter les erreurs de données et les corriger de manière automatique sans intervention humaine.
    • Optimisation des volumes de données : ReFS est conçu pour fonctionner de manière optimale avec des systèmes de stockage à grande échelle, comme les serveurs de fichiers, et dans les environnements de virtualisation.
    • Support natif du stockage des données sur des disques de grande taille et des systèmes de fichiers de plus de 2 To.

Différences entre NTFS et ReFS :

  • NTFS : Plus polyvalent et utilisé pour une gamme plus large d'applications.
  • ReFS : Mieux adapté pour les charges de travail nécessitant une résilience et une intégrité des données maximales, comme les bases de données ou les systèmes de virtualisation.

Choix entre NTFS et ReFS :

  • Utiliser NTFS : Pour les systèmes qui nécessitent des fonctionnalités de sécurité avancées et qui sont moins sensibles à la résilience des données.
  • Utiliser ReFS : Pour les serveurs qui nécessitent une haute disponibilité, une gestion efficace des données et une tolérance aux pannes, comme dans les environnements de stockage de fichiers à grande échelle ou les configurations de virtualisation.

8.3. Mise en place du stockage partagé : iSCSI, SAN, NAS

iSCSI (Internet Small Computer System Interface) :

  • iSCSI est une technologie de stockage qui permet de relier des serveurs à des périphériques de stockage à travers un réseau IP. Contrairement au SAN traditionnel (Storage Area Network) qui utilise des connexions Fibre Channel, iSCSI permet de créer des réseaux de stockage à moindre coût en utilisant des réseaux Ethernet existants.
  • Avantages d’iSCSI :
    • Permet de partager des disques durs ou des volumes sur un réseau IP.
    • Simplifie l'administration du stockage en permettant la gestion des périphériques à distance via des commandes SCSI.
    • Permet une centralisation du stockage pour une gestion plus facile.

SAN (Storage Area Network) :

  • Un SAN est un réseau dédié reliant des serveurs à des périphériques de stockage, utilisant généralement le protocole Fibre Channel ou iSCSI.
  • Avantages d’un SAN :
    • Haute performance et faible latence en raison de l’utilisation de réseaux dédiés.
    • Centralisation et gestion des ressources de stockage.
    • Offre une scalabilité facile pour ajouter des ressources de stockage supplémentaires.

NAS (Network Attached Storage) :

  • Un NAS est un périphérique de stockage en réseau permettant d'accéder aux fichiers partagés via des protocoles comme SMB (Server Message Block) ou NFS (Network File System).
  • Avantages d’un NAS :
    • Offre une solution simple et rentable pour partager des fichiers à travers un réseau.
    • Facile à déployer pour un stockage partagé accessible par plusieurs utilisateurs ou serveurs.
    • Très utilisé dans les environnements de collaboration ou de sauvegarde de fichiers.

Choisir entre iSCSI, SAN et NAS :

  • iSCSI : Idéal pour les environnements où le stockage partagé est nécessaire sans l'infrastructure coûteuse d’un SAN.
  • SAN : Utilisé dans des environnements de production nécessitant une haute performance et une grande capacité de stockage.
  • NAS : Recommandé pour des partages de fichiers simples et la centralisation des données dans des environnements de petites ou moyennes entreprises.

8.4. Gestion des espaces de stockage (Storage Spaces)

Storage Spaces :

Storage Spaces est une fonctionnalité de Windows Server qui permet de créer des volumes de stockage résilients à partir de disques physiques. Elle permet également de regrouper plusieurs disques physiques pour en former un espace de stockage unifié et tolérant aux pannes.

Fonctionnalités principales de Storage Spaces :

  1. Virtualisation du stockage : Crée des espaces de stockage en regroupant plusieurs disques physiques dans une piscine de stockage virtuelle.
  1. Tolérance aux pannes : Propose des options comme la réplication des données (pour la redondance) et des volumes en miroir pour la protection contre les pannes matérielles.
  1. Reconstruction des données : En cas de défaillance d'un disque, Storage Spaces peut reconstruire les données à partir des disques restants.
  1. Allocation flexible : Les volumes peuvent être étendus dynamiquement à mesure que de nouveaux disques sont ajoutés à l’espace de stockage.

Types de configurations Storage Spaces :

  1. Simple : Les données sont réparties sur plusieurs disques sans redondance. Les performances sont élevées, mais il n'y a pas de protection contre la défaillance d’un disque.
  1. Rédundant (Mirror) : Les données sont dupliquées sur plusieurs disques, offrant une protection contre la panne d'un disque.
  1. Parity : Les données et la parité (données de récupération) sont réparties sur plusieurs disques. Cela optimise l’espace tout en offrant une protection contre les pannes.

Gestion des Storage Spaces :

  1. Utiliser PowerShell ou le gestionnaire de serveur pour créer des espaces de stockage.
  1. Contrôler la santé des espaces de stockage et des disques.
  1. Ajouter des disques physiques ou des capacités de stockage supplémentaires à l’espace existant pour garantir l’évolutivité du stockage.
9. Gestion des sauvegardes et de la récupération

9.1. Utilisation de Windows Server Backup pour sauvegarder et restaurer

Windows Server Backup est l'outil intégré dans Windows Server permettant de réaliser des sauvegardes de fichiers, de volumes, et même de systèmes complets. Il offre une solution simple mais robuste pour protéger les données critiques.

Sauvegarde avec Windows Server Backup :

  1. Installation de Windows Server Backup :
    • Windows Server Backup n’est pas installé par défaut dans certaines éditions de Windows Server. Vous pouvez l'installer via le Gestionnaire de serveur sous l'option "Ajouter des rôles et des fonctionnalités".
  1. Types de sauvegardes disponibles :
    • Sauvegarde complète du système : Sauvegarde de l'ensemble du système d'exploitation, des fichiers système et des configurations. Elle permet une récupération complète en cas de sinistre.
    • Sauvegarde de fichiers et dossiers : Sauvegarde sélective de fichiers et répertoires spécifiques.
    • Sauvegarde de volumes : Sauvegarde de disques ou volumes spécifiques (y compris les volumes NTFS).
    • Sauvegarde programmée : Vous pouvez configurer des sauvegardes automatiques, quotidiennes ou hebdomadaires, pour garantir que les données sont sauvegardées régulièrement.
  1. Processus de sauvegarde :
    • Ouvrir Windows Server Backup dans le Gestionnaire de serveur ou via l’outil de ligne de commande.
    • Sélectionner les éléments à sauvegarder (disques, volumes, dossiers, ou une sauvegarde complète du système).
    • Choisir l'emplacement de la sauvegarde (disque dur local, lecteur réseau ou autre média externe).
  1. Restauration avec Windows Server Backup :
    • Vous pouvez restaurer des fichiers spécifiques ou effectuer une restauration complète du système à partir des sauvegardes créées.
    • La restauration du système peut inclure la récupération des paramètres système et des configurations d’applications, ce qui permet de restaurer le serveur dans son état d'origine.

Limites de Windows Server Backup :

  • Ce système n'inclut pas la prise en charge des sauvegardes en continu ou des sauvegardes sur disque réseau de manière native, et n'offre pas la même granularité de récupération que des solutions plus spécialisées.

9.2. Plan de reprise après sinistre (Disaster Recovery)

Un plan de reprise après sinistre (DRP - Disaster Recovery Plan) est essentiel pour garantir que l'infrastructure et les données critiques d'une organisation puissent être récupérées après un incident majeur (panne de serveur, défaillance matérielle, cyberattaque, etc.).

Principaux éléments du plan de reprise après sinistre :

  1. Évaluation des risques : Identifier les risques potentiels pour les systèmes informatiques, les applications, et les données. Cela inclut les pannes matérielles, les erreurs humaines, les cyberattaques, et les catastrophes naturelles.
  1. Stratégies de sauvegarde et de récupération :
    • Redondance : Utiliser des solutions de stockage redondantes, comme les volumes RAID, les sauvegardes externes et les réplications, pour garantir que les données peuvent être récupérées en cas de panne.
    • Répartition géographique des sauvegardes : Les données doivent être sauvegardées dans des sites géographiquement distants pour se protéger contre des événements locaux tels qu’un incendie ou une inondation.
  1. Tests de récupération : Les plans de récupération doivent être testés régulièrement pour garantir leur efficacité. Ces tests incluent la récupération des systèmes critiques dans un délai déterminé.
  1. Communication : Établir une communication claire avec tous les acteurs du plan (équipes IT, direction, employés) pour garantir une coordination rapide en cas de sinistre.
  1. Récupération de la continuité des affaires : Dans certains cas, l'organisation peut mettre en œuvre des solutions comme la virtualisation des serveurs pour permettre une reprise plus rapide des activités.

Outils de reprise après sinistre sous Windows Server :

  • Failover Clustering : Permet de configurer des clusters de serveurs pour assurer une haute disponibilité et faciliter la reprise rapide des services en cas de défaillance d'un serveur.
  • Windows Server Backup : Permet de sauvegarder et de restaurer l'ensemble du système d'exploitation, offrant ainsi une solution de récupération après sinistre.
  • Hyper-V Replica : Utilisé pour répliquer des machines virtuelles entre différents sites, permettant une reprise rapide en cas de défaillance du site principal.

9.3. Configuration de points de restauration système

Les points de restauration système permettent de créer des snapshots du système d’exploitation à des moments spécifiques. Ils sont utilisés principalement pour la récupération du système en cas de problèmes (pannes logicielles, conflits, etc.) sans affecter les données des utilisateurs.

Création d’un point de restauration :

  1. Activer la protection du système : Dans Propriétés système, sous l’onglet "Protection du système", vous pouvez activer la protection du système pour les disques système.
  1. Créer un point de restauration manuellement :
    • Ouvrir le Panneau de configuration, puis Système et sécurité > Système > Protection du système.
    • Cliquez sur "Créer" pour créer un nouveau point de restauration.
  1. Utilisation de la restauration :
    • Si un problème survient, vous pouvez utiliser la fonctionnalité de Restauration du système pour revenir à un état stable antérieur du système sans affecter les données des utilisateurs.
    • Ce processus est particulièrement utile pour résoudre des problèmes liés aux installations de nouveaux logiciels ou aux mises à jour qui perturbent le fonctionnement normal du système.

Limites des points de restauration :

  • Les points de restauration ne sauvegardent pas les fichiers personnels de l'utilisateur, ni les fichiers générés par des applications.
  • Ils ne sont pas une solution de sauvegarde complète et doivent être utilisés en complément d'autres stratégies de sauvegarde.

9.4. Introduction à Azure Backup et la sauvegarde dans le Cloud

Azure Backup est un service de sauvegarde basé sur le cloud proposé par Microsoft. Il permet de sauvegarder les données locales sur des serveurs Windows vers le cloud Azure. Ce service est particulièrement utile pour les organisations cherchant à renforcer leur stratégie de sauvegarde tout en garantissant une récupération rapide et fiable des données.

Avantages de Azure Backup :

  1. Sauvegarde dans le cloud : Permet de sauvegarder les données hors site, ce qui protège contre les risques locaux (incendies, inondations, vol, etc.).
  1. Évolutivité : Le stockage des sauvegardes est pratiquement illimité, et vous payez uniquement pour l’espace utilisé, ce qui offre une grande flexibilité.
  1. Sécurité : Les données sont chiffrées lors de la sauvegarde et du transfert vers le cloud. Azure Backup prend en charge des mécanismes de sécurité avancés comme l’authentification multi-facteurs.
  1. Automatisation : La sauvegarde peut être automatisée selon un calendrier défini, ce qui réduit le risque d’oubli.
  1. Récupération rapide : Azure Backup permet de restaurer des fichiers individuels, des applications ou même des systèmes entiers dans un délai rapide.

Configurer Azure Backup :

  1. Préparer le compte Azure : Créez un compte Azure et configurez un coffre de sauvegarde dans le portail Azure.
  1. Installer l'agent de sauvegarde Azure sur le serveur Windows.
  1. Configurer la sauvegarde : Définissez les éléments à sauvegarder (fichiers, volumes, bases de données, etc.) et le calendrier de sauvegarde.
  1. Restaurer les données : En cas de besoin, les données peuvent être restaurées à partir de votre coffre de sauvegarde Azure, avec des options de restauration granulaire.
10. Automatisation et gestion avec PowerShell

10.1. Introduction à PowerShell : commandes de base, scripts simples

PowerShell est un environnement de ligne de commande et un langage de script orienté objet utilisé pour l'administration système sur Windows. Il permet aux administrateurs d'automatiser des tâches répétitives et de configurer des serveurs de manière cohérente et efficace.

Commandes de base de PowerShell :

  1. Cmdlets : PowerShell utilise des cmdlets (commandlets) pour effectuer des actions spécifiques. Les cmdlets sont de petites unités de commande en ligne conçues pour être utilisées dans des scripts ou manuellement.
    • Exemple : Get-Help (affiche l’aide), Get-Command (affiche toutes les commandes disponibles).
  1. Manipulation des objets :
    • PowerShell traite les données sous forme d’objets, contrairement aux simples chaînes de texte. Cela permet de manipuler des données complexes.
    • Exemple : Get-Service renvoie des objets représentant les services en cours d'exécution, que vous pouvez filtrer ou manipuler (ex. : Get-Service | Where-Object { $_.Status -eq 'Running' }).
  1. Variables : Les variables sont définies avec un signe $. Par exemple, $serverName = "Server01".
  1. Exemples de commandes de base :
    • Get-Process : Liste des processus en cours.
    • Set-ExecutionPolicy RemoteSigned : Modifie la politique d'exécution des scripts PowerShell.
    • Get-EventLog -LogName System : Affiche les événements du journal système.

Écrire un script PowerShell simple :

Voici un exemple de script simple qui redémarre un service spécifique : 

$serviceName = "wuauserv"  # Nom du service à redémarrer
$service = Get-Service -Name $serviceName

if ($service.Status -eq "Running") {
    Restart-Service -Name $serviceName
    Write-Output "$serviceName a été redémarré."
} else {
    Write-Output "$serviceName n'est pas en cours d'exécution."
}

10.2. Administration avancée avec PowerShell : GPO, AD, et Hyper-V

PowerShell permet une gestion avancée de l'infrastructure Windows Server, y compris la gestion des Stratégies de groupe (GPO), des Utilisateurs et groupes Active Directory (AD), et des machines virtuelles Hyper-V.

Gestion des GPO avec PowerShell :

Les cmdlets PowerShell pour GPO permettent de configurer et gérer les stratégies de groupe directement depuis la ligne de commande.

10.3. Utilisation de PowerShell Desired State Configuration (DSC)

PowerShell Desired State Configuration (DSC) est une fonctionnalité de PowerShell permettant de définir, de configurer et de gérer l'état des serveurs. Avec DSC, les administrateurs peuvent spécifier l’état souhaité d'un serveur (ou d'un groupe de serveurs), et PowerShell s'assurera que l'état souhaité est appliqué.

Concepts clés de DSC :

  • Configuration : Une configuration DSC définit l'état souhaité d’un ou plusieurs serveurs.
  • Ressources : Ce sont des éléments utilisés dans une configuration pour définir l'état de la machine (par exemple, la présence d'un fichier, un paramètre de registre, etc.).
  • Application de configuration : Après avoir défini la configuration, vous pouvez l'appliquer à un ou plusieurs serveurs via PowerShell.

Exemple d'une configuration DSC :

Voici un exemple simple qui s'assure qu’un fichier spécifique existe sur le serveur. 

Configuration EnsureFileExists {
    Node "localhost" {
        File MyFile {
            DestinationPath = "C:\Temp\myfile.txt"
            SourcePath      = "C:\Scripts\myfile.txt"
            Ensure          = "Present"
        }
    }
}

EnsureFileExists
Start-DscConfiguration -Path "C:\DSC\EnsureFileExists" -Wait -Force

Dans cet exemple, DSC s’assure que le fichier myfile.txt est présent dans le répertoire C:\Temp. Si le fichier n’existe pas, DSC le copie depuis le chemin source.

15.2. Microsoft Certified: Azure Administrator Associate

Cette certification se concentre sur la gestion des solutions cloud avec Azure.

Exigences

Pour devenir un Azure Administrator Associate, il faut réussir l’examen AZ-104.

Compétences clés évaluées

  • Gestion des identités (Azure AD).
  • Mise en œuvre des ressources virtuelles (machines virtuelles, stockage, réseaux).
  • Surveillance des performances et optimisation des coûts.
  • Gestion de la sauvegarde, de la récupération et de la sécurité des données.

Applications pratiques

  • Administration des environnements hybrides.
  • Migration des charges de travail vers Azure.
  • Optimisation des performances cloud pour des solutions d'entreprise.
