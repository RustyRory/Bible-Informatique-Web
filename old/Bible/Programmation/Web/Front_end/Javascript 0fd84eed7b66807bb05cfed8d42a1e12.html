Javascript

1. Fondamentaux du langage

1.1. Types de données et variables

Voici un résumé plus détaillé des types de données et variables en JavaScript :

1.1.1. Types de données primitifs

1.1.1.1 Number

Le type Number en JavaScript représente à la fois les nombres entiers et à virgule flottante.

  • Entiers : let age = 25;
  • Flottants : let pi = 3.14159;
  • Notation scientifique : let avogadro = 6.022e23;
  • Valeurs spéciales : Infinity, -Infinity, et NaN (Not a Number)

JavaScript utilise le standard IEEE 754 pour représenter les nombres, ce qui peut parfois conduire à des imprécisions dans les calculs avec des décimales.

1.1.1.2. String

Les chaînes de caractères peuvent être déclarées avec des guillemets simples, doubles ou backticks (template literals).

  • Guillemets simples : let prenom = 'Alice';
  • Guillemets doubles : let nom = "Dupont";
  • Backticks (ES6+) : let message = `Bonjour ${prenom}`;

Les template literals permettent l'interpolation de variables et les chaînes multi-lignes.

1.1.1.3. Boolean

Le type Boolean ne peut avoir que deux valeurs : true ou false.

  • let estMajeur = true;
  • let aDesEnfants = false;

Les opérations de comparaison renvoient des booléens : let estPlusGrand = 5 > 3; // true

1.1.1.4. Undefined

Une variable déclarée mais non initialisée a la valeur undefined.

  • let maVariable;

On peut aussi assigner explicitement undefined : let x = undefined;

1.1.1.5. Null

Null représente l'absence intentionnelle de toute valeur d'objet.

  • let donnees = null;

Null est souvent utilisé pour réinitialiser une variable ou indiquer qu'un objet n'existe pas.

1.1.1.6. Symbol (ES6+)

Les symboles sont des identifiants uniques, principalement utilisés comme clés d'objets.

  • const monSymbole = Symbol('description');
  • const autreSymbole = Symbol('description');

Même avec la même description, deux symboles sont toujours différents : monSymbole !== autreSymbole

1.1.1.7. BigInt (ES11+)

BigInt permet de représenter des entiers de taille arbitraire.

  • const grandNombre = 1234567890123456789012345678901234567890n;

On peut aussi créer un BigInt avec la fonction BigInt() : const autreBigInt = BigInt("1234567890");

1.1.2. Types de données complexes

1.1.2.1. Object

Les objets sont des collections de paires clé-valeur. 

let personne = {
  nom: "Dupont",
  age: 30,
  adresse: {
    rue: "123 Rue Principale",
    ville: "Paris"
  }
};

On accède aux propriétés avec la notation point ou crochet : personne.nom ou personne["nom"]

1.1.2.2. Array

Les tableaux sont des objets spéciaux pour stocker des listes ordonnées de valeurs. 

let fruits = ["pomme", "banane", "orange"];
let mixte = [1, "deux", { trois: 3 }, [4, 5]];

On accède aux éléments par leur index : fruits[0] (les index commencent à 0)

1.1.2.3. Function

Les fonctions sont des objets de première classe en JavaScript. 

function direBonjour(nom) {
  return `Bonjour ${nom}!`;
}

let saluer = function(nom) {
  console.log(`Salut ${nom}!`);
};

const flechee = (x) => x * 2;

Les fonctions peuvent être assignées à des variables, passées comme arguments, et retournées par d'autres fonctions.

1.1.3. Déclaration de variables

1.1.3.1. var

Déclaration de variable avec une portée de fonction ou globale.

  • var x = 5;

Les variables déclarées avec var sont hissées (hoisted) au début de leur portée.

1.1.3.2. let (ES6+)

Déclaration de variable avec une portée de bloc.

  • let y = 10;

let permet une meilleure gestion de la portée et évite les problèmes liés au hoisting.

1.1.3.3. const (ES6+)

Déclaration de constante (ne peut pas être réassignée).

  • const PI = 3.14159;

Pour les objets et tableaux, const empêche la réassignation de la variable, mais pas la modification de son contenu.

Il est important de noter que JavaScript est un langage à typage dynamique et faible. Cela signifie que les variables peuvent changer de type au cours de l'exécution du programme, et que JavaScript effectue des conversions de type implicites dans certaines situations.

Exemple de typage dynamique : 

let x = 5;       // x est un nombre
x = "cinq";      // x est maintenant une chaîne de caractères
x = true;        // x est maintenant un booléen

Exemple de conversion de type implicite : 

console.log("5" + 3);     // Affiche "53" (3 est converti en chaîne)
console.log("5" - 3);     // Affiche 2 ("5" est converti en nombre)
console.log("5" == 5);    // Affiche true (comparaison avec conversion de type)
console.log("5" === 5);   // Affiche false (comparaison stricte sans conversion)

En conclusion, une bonne compréhension des types de données et des variables en JavaScript est essentielle pour écrire du code efficace et éviter les erreurs courantes liées au typage dynamique.

1.2. Structures de contrôle (if, else, switch, boucles)

Les structures de contrôle en JavaScript permettent de gérer le flux d'exécution du code. Voici un résumé détaillé des principales structures de contrôle :

1.2.1. Structures conditionnelles

1.2.1.1. if...else

La structure if...else permet d'exécuter du code en fonction d'une condition. 

let age = 18;
if (age >= 18) {
    console.log("Vous êtes majeur");
} else {
    console.log("Vous êtes mineur");
}

1.2.1.2. else if

On peut ajouter des conditions supplémentaires avec else if. 

let note = 75;
if (note >= 90) {
    console.log("Excellent");
} else if (note >= 70) {
    console.log("Bien");
} else if (note >= 50) {
    console.log("Passable");
} else {
    console.log("Échec");
}

1.2.1.3. switch

La structure switch est utile pour comparer une valeur à plusieurs cas possibles. 

let jour = "Lundi";
switch (jour) {
    case "Lundi":
        console.log("Début de semaine");
        break;
    case "Vendredi":
        console.log("Fin de semaine");
        break;
    default:
        console.log("Milieu de semaine");
}

1.2.2. Boucles

1.2.2.1. for

La boucle for est utilisée pour répéter une action un nombre défini de fois. 

for (let i = 0; i < 5; i++) {
    console.log(i);
}

1.2.2.2. while

La boucle while répète une action tant qu'une condition est vraie. 

let i = 0;
while (i < 5) {
    console.log(i);
    i++;
}

1.2.2.3. do...while

Similaire à while, mais garantit que le code s'exécute au moins une fois. 

let i = 0;
do {
    console.log(i);
    i++;
} while (i < 5);

1.2.2.4. for...in

Utilisée pour itérer sur les propriétés énumérables d'un objet. 

let personne = {nom: "Dupont", age: 30, ville: "Paris"};
for (let prop in personne) {
    console.log(prop + ": " + personne[prop]);
}

1.2.2.5. for...of (ES6+)

Utilisée pour itérer sur les éléments d'une collection itérable (comme un tableau ou une chaîne). 

let fruits = ["pomme", "banane", "orange"];
for (let fruit of fruits) {
    console.log(fruit);
}

1.2.3. Contrôle de boucle

1.2.3.1. break

L'instruction break permet de sortir immédiatement d'une boucle ou d'un switch. 

for (let i = 0; i < 10; i++) {
    if (i === 5) break;
    console.log(i);
}

1.2.3.2. continue

L'instruction continue passe à l'itération suivante de la boucle. 

for (let i = 0; i < 5; i++) {
    if (i === 2) continue;
    console.log(i);
}

Ces structures de contrôle sont essentielles pour créer des programmes JavaScript dynamiques et réactifs. Elles permettent de gérer efficacement le flux d'exécution du code en fonction des conditions et des besoins spécifiques de l'application.

1.3. Fonctions et portée

Les fonctions et la portée sont des concepts fondamentaux en JavaScript. Voici un résumé détaillé :

1.3.1. Définition et déclaration de fonctions

Il existe plusieurs façons de définir des fonctions en JavaScript :

1.3.1.1. Déclaration de fonction 

function saluer(nom) {
    return `Bonjour, ${nom}!`;
}
console.log(saluer("Alice")); // Affiche: Bonjour, Alice!

1.3.1.2. Expression de fonction 

const saluer = function(nom) {
    return `Bonjour, ${nom}!`;
};
console.log(saluer("Bob")); // Affiche: Bonjour, Bob!

1.3.1.3. Fonction fléchée (ES6+) 

const saluer = (nom) => `Bonjour, ${nom}!`;
console.log(saluer("Charlie")); // Affiche: Bonjour, Charlie!

1.3.2. Paramètres et arguments

Les fonctions peuvent accepter des paramètres et utiliser des valeurs par défaut : 

function multiplier(a, b = 1) {
    return a * b;
}
console.log(multiplier(5));    // Affiche: 5
console.log(multiplier(5, 3)); // Affiche: 15

1.3.3. Portée des variables

La portée détermine la visibilité et l'accessibilité des variables dans le code.

1.3.3.1. Portée globale 

var globale = "Je suis globale";

function afficherGlobale() {
    console.log(globale);
}

afficherGlobale(); // Affiche: Je suis globale

1.3.3.2. Portée locale (fonction) 

function exemple() {
    var locale = "Je suis locale";
    console.log(locale);
}

exemple(); // Affiche: Je suis locale
// console.log(locale); // Erreur: locale is not defined

1.3.3.3. Portée de bloc (let et const) 

if (true) {
    let blocVariable = "Je suis dans un bloc";
    console.log(blocVariable); // Affiche: Je suis dans un bloc
}
// console.log(blocVariable); // Erreur: blocVariable is not defined

1.3.4. Closure

Une closure est une fonction qui a accès aux variables de sa portée extérieure : 

function creerCompteur() {
    let compte = 0;
    return function() {
        return ++compte;
    };
}

const compteur = creerCompteur();
console.log(compteur()); // Affiche: 1
console.log(compteur()); // Affiche: 2

1.3.5. This et liaison de contexte

Le mot-clé 'this' fait référence au contexte d'exécution d'une fonction : 

const personne = {
    nom: "Alice",
    saluer: function() {
        console.log(`Bonjour, je m'appelle ${this.nom}`);
    }
};

personne.saluer(); // Affiche: Bonjour, je m'appelle Alice

const saluerHorsDuContexte = personne.saluer;
// saluerHorsDuContexte(); // Erreur: this.nom est undefined

const saluerLie = personne.saluer.bind(personne);
saluerLie(); // Affiche: Bonjour, je m'appelle Alice

En comprenant ces concepts, vous pouvez écrire du code JavaScript plus propre, plus modulaire et plus efficace.

1.4. Objets et tableaux

Les objets et les tableaux sont des structures de données fondamentales en JavaScript. Voici un résumé détaillé :

1.4.1. Objets

Les objets en JavaScript sont des collections de paires clé-valeur.

1.4.1.1. Création d'objets 

// Notation littérale
let personne = {
    nom: "Dupont",
    age: 30,
    ville: "Paris"
};

// Constructeur Object
let voiture = new Object();
voiture.marque = "Renault";
voiture.modele = "Clio";

// Fonction constructeur
function Animal(espece, nom) {
    this.espece = espece;
    this.nom = nom;
}
let chien = new Animal("Canin", "Rex");

1.4.1.2. Accès aux propriétés 

console.log(personne.nom); // Notation point
console.log(personne["age"]); // Notation crochet

// Vérification de l'existence d'une propriété
console.log("nom" in personne); // true
console.log(personne.hasOwnProperty("adresse")); // false

1.4.1.3. Méthodes d'objet 

let calculatrice = {
    addition: function(a, b) {
        return a + b;
    },
    soustraction: (a, b) => a - b // Syntaxe de fonction fléchée
};

console.log(calculatrice.addition(5, 3)); // 8

1.4.2. Tableaux

Les tableaux sont des objets spéciaux utilisés pour stocker des listes ordonnées d'éléments.

1.4.2.1. Création de tableaux 

// Notation littérale
let fruits = ["pomme", "banane", "orange"];

// Constructeur Array
let nombres = new Array(1, 2, 3, 4, 5);

// Array.from (ES6+)
let lettres = Array.from("ABCDE");

1.4.2.2. Accès et modification des éléments 

console.log(fruits[0]); // "pomme"
fruits[1] = "kiwi";
console.log(fruits); // ["pomme", "kiwi", "orange"]

// Ajout et suppression d'éléments
fruits.push("mangue"); // Ajoute à la fin
fruits.unshift("fraise"); // Ajoute au début
let dernier = fruits.pop(); // Supprime et retourne le dernier élément
let premier = fruits.shift(); // Supprime et retourne le premier élément

1.4.2.3. Méthodes de tableau courantes 

// forEach
fruits.forEach(fruit => console.log(fruit));

// map
let longueurs = fruits.map(fruit => fruit.length);

// filter
let longsFruits = fruits.filter(fruit => fruit.length > 5);

// reduce
let total = nombres.reduce((acc, val) => acc + val, 0);

// find
let premierLong = fruits.find(fruit => fruit.length > 5);

// some et every
let quelquesLongs = fruits.some(fruit => fruit.length > 5);
let tousLongs = fruits.every(fruit => fruit.length > 5);

1.4.2.4. Spread operator et déstructuration (ES6+) 

// Spread operator
let plusDeFruits = [...fruits, "papaye", "ananas"];

// Déstructuration
let [fruit1, fruit2, ...reste] = fruits;
console.log(fruit1, fruit2, reste);

En maîtrisant ces concepts d'objets et de tableaux, vous serez capable de manipuler efficacement les données dans vos applications JavaScript.

1.5. Prototypes et héritage

Les prototypes et l'héritage sont des concepts fondamentaux en JavaScript qui permettent de créer des objets basés sur d'autres objets. Voici un résumé détaillé :

1.5.1. Prototypes

En JavaScript, chaque objet a un prototype interne, qui est lui-même un objet. Ce prototype sert de "modèle" pour l'objet, fournissant des propriétés et des méthodes que l'objet peut utiliser. 

// Création d'un objet prototype
let animalPrototype = {
    respirer: function() {
        console.log("Je respire");
    }
};

// Création d'un objet basé sur le prototype
let chien = Object.create(animalPrototype);
chien.aboyer = function() {
    console.log("Woof!");
};

chien.respirer(); // Affiche: Je respire
chien.aboyer();   // Affiche: Woof!

1.5.2. Chaîne de prototypes

Les objets en JavaScript forment une chaîne de prototypes. Si une propriété n'est pas trouvée sur un objet, JavaScript la recherche dans son prototype, puis dans le prototype du prototype, et ainsi de suite. 

let mammiferePrototype = Object.create(animalPrototype);
mammiferePrototype.allaiter = function() {
    console.log("J'allaite mes petits");
};

let chien = Object.create(mammiferePrototype);
chien.aboyer = function() {
    console.log("Woof!");
};

chien.respirer(); // Affiche: Je respire (hérité d'animalPrototype)
chien.allaiter(); // Affiche: J'allaite mes petits (hérité de mammiferePrototype)
chien.aboyer();   // Affiche: Woof! (propre méthode)

1.5.3. Constructeurs et l'opérateur new

Les fonctions constructeurs sont utilisées avec l'opérateur new pour créer des objets avec un prototype spécifique. 

function Animal(nom) {
    this.nom = nom;
}

Animal.prototype.respirer = function() {
    console.log(this.nom + " respire");
};

let chat = new Animal("Minou");
chat.respirer(); // Affiche: Minou respire

1.5.4. Héritage

L'héritage en JavaScript se fait via la chaîne de prototypes. On peut créer une "classe" enfant qui hérite d'une "classe" parent. 

function Chien(nom, race) {
    Animal.call(this, nom); // Appel du constructeur parent
    this.race = race;
}

// Héritage du prototype
Chien.prototype = Object.create(Animal.prototype);
Chien.prototype.constructor = Chien;

Chien.prototype.aboyer = function() {
    console.log(this.nom + " aboie");
};

let rex = new Chien("Rex", "Berger Allemand");
rex.respirer(); // Affiche: Rex respire
rex.aboyer();   // Affiche: Rex aboie

1.5.5. Classes ES6+

ES6 a introduit une syntaxe de classe plus familière pour gérer les prototypes et l'héritage, bien que le mécanisme sous-jacent reste basé sur les prototypes. 

class Animal {
    constructor(nom) {
        this.nom = nom;
    }

    respirer() {
        console.log(this.nom + " respire");
    }
}

class Chien extends Animal {
    constructor(nom, race) {
        super(nom);
        this.race = race;
    }

    aboyer() {
        console.log(this.nom + " aboie");
    }
}

let max = new Chien("Max", "Labrador");
max.respirer(); // Affiche: Max respire
max.aboyer();   // Affiche: Max aboie

En comprenant ces concepts, vous pouvez créer des hiérarchies d'objets complexes et réutiliser efficacement le code dans vos applications JavaScript.

2. Concepts avancés

2.1. Closures

Les closures (fermetures) sont un concept fondamental en JavaScript qui permet à une fonction de "se souvenir" de son environnement lexical, même lorsqu'elle est exécutée en dehors de cet environnement. Voici un résumé détaillé :

2.1.1. Définition

Une closure est créée lorsqu'une fonction est définie à l'intérieur d'une autre fonction. Elle permet à la fonction interne d'accéder aux variables de la fonction externe, même après que cette dernière ait terminé son exécution.

2.1.2. Caractéristiques principales

  • Accès aux variables de la portée parente
  • Préservation de l'état
  • Encapsulation des données

2.1.3. Exemples

2.1.3.1. Création d'une closure simple 

function creerCompteur() {
    let count = 0;
    return function() {
        count++;
        console.log(count);
    };
}

const compteur = creerCompteur();
compteur(); // Affiche: 1
compteur(); // Affiche: 2
compteur(); // Affiche: 3

Dans cet exemple, la fonction interne a accès à la variable 'count' de la fonction externe, même après que 'creerCompteur' ait terminé son exécution.

2.1.3.2. Utilisation pour créer des fonctions privées 

function creerCalculateur() {
    let total = 0;
    
    function ajouter(x) {
        total += x;
    }
    
    function soustraire(x) {
        total -= x;
    }
    
    function obtenirTotal() {
        return total;
    }
    
    return {
        ajouter,
        soustraire,
        obtenirTotal
    };
}

const calc = creerCalculateur();
calc.ajouter(5);
calc.soustraire(2);
console.log(calc.obtenirTotal()); // Affiche: 3

Ici, 'total' est une variable privée accessible uniquement via les méthodes exposées.

2.1.3.3. Création de fonctions avec des paramètres prédéfinis 

function multiplier(x) {
    return function(y) {
        return x * y;
    };
}

const doubler = multiplier(2);
const tripler = multiplier(3);

console.log(doubler(5)); // Affiche: 10
console.log(tripler(5)); // Affiche: 15

Cette technique, appelée "currying", utilise les closures pour créer des fonctions spécialisées.

2.1.4. Avantages des closures

  • Encapsulation des données
  • Création de fonctions d'ordre supérieur
  • Implémentation de modules

2.1.5. Points à considérer

  • Les closures peuvent consommer plus de mémoire car elles maintiennent une référence à leur environnement lexical
  • Une utilisation excessive peut rendre le code plus difficile à comprendre

En maîtrisant les closures, vous pouvez écrire un code JavaScript plus propre, plus modulaire et plus puissant.

2.2. Programmation fonctionnelle

La programmation fonctionnelle est un paradigme de programmation qui traite le calcul comme l'évaluation de fonctions mathématiques et évite de changer l'état et les données mutables. Voici un résumé détaillé de la programmation fonctionnelle en JavaScript :

2.2.1. Principes fondamentaux

  • Immutabilité : Les données ne changent pas une fois créées.
  • Fonctions pures : Les fonctions produisent toujours le même résultat pour les mêmes entrées, sans effets secondaires.
  • Récursivité : Utilisation de la récursion plutôt que des boucles pour des opérations répétitives.
  • Fonctions d'ordre supérieur : Les fonctions peuvent prendre d'autres fonctions comme arguments ou les retourner.

2.2.2. Exemples de concepts clés

2.2.2.1. Immutabilité 

// Mauvaise pratique (mutation)
const arr = [1, 2, 3];
arr.push(4);

// Bonne pratique (immutabilité)
const arr = [1, 2, 3];
const newArr = [...arr, 4];

2.2.2.2. Fonctions pures 

// Fonction pure
function add(a, b) {
    return a + b;
}

// Fonction impure (dépend d'un état externe)
let total = 0;
function addToTotal(value) {
    total += value;
    return total;
}

2.2.2.3. Fonctions d'ordre supérieur 

const numbers = [1, 2, 3, 4, 5];

// map est une fonction d'ordre supérieur
const doubled = numbers.map(num => num * 2);

// Création d'une fonction d'ordre supérieur
function multiplyBy(factor) {
    return function(number) {
        return number * factor;
    }
}

const double = multiplyBy(2);
console.log(double(5)); // Affiche: 10

2.2.3. Techniques de programmation fonctionnelle

2.2.3.1. Composition de fonctions 

const compose = (f, g) => x => f(g(x));

const addOne = x => x + 1;
const double = x => x * 2;

const addOneThenDouble = compose(double, addOne);

console.log(addOneThenDouble(3)); // Affiche: 8

2.2.3.2. Currying 

const multiply = (a, b) => a * b;
const curriedMultiply = a => b => a * b;

console.log(multiply(2, 3)); // Affiche: 6
console.log(curriedMultiply(2)(3)); // Affiche: 6

const double = curriedMultiply(2);
console.log(double(4)); // Affiche: 8

2.2.3.3. Récursion 

function factorial(n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

console.log(factorial(5)); // Affiche: 120

2.2.4. Avantages de la programmation fonctionnelle

  • Code plus prévisible et plus facile à tester
  • Réduction des effets secondaires
  • Meilleure modularité et réutilisabilité du code
  • Facilite le parallélisme et la concurrence

2.2.5. Bibliothèques pour la programmation fonctionnelle en JavaScript

  • Lodash/FP : Version fonctionnelle de Lodash
  • Ramda : Bibliothèque conçue spécifiquement pour la programmation fonctionnelle
  • Immutable.js : Pour travailler avec des structures de données immuables

En adoptant ces principes et techniques de programmation fonctionnelle, vous pouvez écrire un code JavaScript plus robuste, plus maintenable et plus expressif.

2.3. Promesses et asynchronicité

Les promesses et l'asynchronicité sont des concepts fondamentaux en JavaScript pour gérer les opérations non bloquantes. Voici un résumé détaillé :

2.3.1. Asynchronicité en JavaScript

JavaScript est un langage à thread unique, mais il peut gérer des opérations asynchrones grâce à la boucle d'événements (event loop). Les opérations asynchrones permettent d'exécuter du code sans bloquer l'exécution du reste du programme.

2.3.2. Callbacks

Historiquement, les callbacks étaient la principale méthode pour gérer l'asynchronicité : 

function fetchData(callback) {
    setTimeout(() => {
        callback('Données récupérées');
    }, 2000);
}

fetchData((data) => {
    console.log(data);
});
console.log('Fin du script');

// Affiche:
// Fin du script
// Données récupérées (après 2 secondes)

Cependant, les callbacks peuvent mener à des structures de code complexes, souvent appelées "callback hell".

2.3.3. Promesses

Les promesses offrent une meilleure façon de gérer les opérations asynchrones : 

const fetchData = new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve('Données récupérées');
        // En cas d'erreur : reject(new Error('Erreur lors de la récupération'));
    }, 2000);
});

fetchData
    .then(data => console.log(data))
    .catch(error => console.error(error));

console.log('Fin du script');

// Affiche:
// Fin du script
// Données récupérées (après 2 secondes)

2.3.4. Chaînage des promesses

Les promesses peuvent être chaînées pour gérer des séquences d'opérations asynchrones : 

function fetchUser() {
    return new Promise(resolve => {
        setTimeout(() => resolve({ id: 1, name: 'Alice' }), 1000);
    });
}

function fetchUserPosts(userId) {
    return new Promise(resolve => {
        setTimeout(() => resolve(['Post 1', 'Post 2']), 1000);
    });
}

fetchUser()
    .then(user => fetchUserPosts(user.id))
    .then(posts => console.log(posts))
    .catch(error => console.error(error));

2.3.5. Async/Await

ES2017 a introduit async/await, une syntaxe plus intuitive pour travailler avec les promesses : 

async function fetchUserAndPosts() {
    try {
        const user = await fetchUser();
        const posts = await fetchUserPosts(user.id);
        console.log(posts);
    } catch (error) {
        console.error(error);
    }
}

fetchUserAndPosts();

2.3.6. Promesses parallèles

Pour exécuter plusieurs promesses en parallèle : 

const promise1 = Promise.resolve('Résultat 1');
const promise2 = new Promise(resolve => setTimeout(() => resolve('Résultat 2'), 2000));

Promise.all([promise1, promise2])
    .then(results => console.log(results))
    .catch(error => console.error(error));

// Affiche: ['Résultat 1', 'Résultat 2'] après 2 secondes

2.3.7. Gestion des erreurs

Les promesses et async/await offrent une meilleure gestion des erreurs : 

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => reject(new Error('Erreur réseau')), 2000);
    });
}

async function getData() {
    try {
        const data = await fetchData();
        console.log(data);
    } catch (error) {
        console.error('Erreur attrapée:', error.message);
    }
}

getData();
// Affiche: Erreur attrapée: Erreur réseau (après 2 secondes)

En maîtrisant ces concepts, vous pouvez écrire du code asynchrone plus propre, plus lisible et plus facile à maintenir en JavaScript.

2.4. Modules ES6+

Les modules ES6+ sont une fonctionnalité importante de JavaScript moderne, permettant une meilleure organisation et réutilisation du code. Voici un résumé détaillé :

2.4.1. Définition et exportation de modules

Les modules permettent d'encapsuler du code et de l'exporter pour l'utiliser dans d'autres fichiers : 

// math.js
export const PI = 3.14159;

export function add(a, b) {
    return a + b;
}

export default function multiply(a, b) {
    return a * b;
}

2.4.2. Importation de modules

Il existe plusieurs façons d'importer des modules : 

// app.js
import multiply, { PI, add } from './math.js';

console.log(PI);  // 3.14159
console.log(add(2, 3));  // 5
console.log(multiply(4, 5));  // 20

2.4.3. Importation avec alias

On peut utiliser des alias pour renommer les imports : 

import { add as addition } from './math.js';
console.log(addition(2, 3));  // 5

2.4.4. Importation de tous les exports

On peut importer tous les exports d'un module sous un namespace : 

import * as MathModule from './math.js';
console.log(MathModule.PI);  // 3.14159
console.log(MathModule.add(2, 3));  // 5

2.4.5. Réexportation

On peut réexporter des modules importés : 

// utils.js
export { add, multiply } from './math.js';
export { default as fetchData } from './api.js';

2.4.6. Importations dynamiques

ES2020 a introduit les importations dynamiques pour charger des modules de manière conditionnelle : 

if (condition) {
    import('./module.js')
        .then(module => {
            // Utiliser le module
        })
        .catch(error => {
            console.error('Erreur de chargement du module:', error);
        });
}

2.4.7. Avantages des modules ES6+

  • Meilleure organisation du code
  • Encapsulation et réduction des conflits de noms
  • Chargement plus efficace des dépendances
  • Support natif dans les navigateurs modernes

Les modules ES6+ offrent une approche puissante et flexible pour structurer les applications JavaScript, améliorant la maintenabilité et la réutilisabilité du code.

2.5. Gestion des erreurs

La gestion des erreurs est un aspect crucial du développement JavaScript. Voici un résumé détaillé avec des exemples :

2.5.1. Try...catch

La structure try...catch est la base de la gestion des erreurs en JavaScript : 

try {
    // Code susceptible de générer une erreur
    let x = y + 1; // y n'est pas défini
} catch (error) {
    console.error("Une erreur s'est produite :", error.message);
}
// Le programme continue son exécution

2.5.2. Finally

Le bloc finally s'exécute toujours, qu'une erreur se produise ou non : 

try {
    // Code susceptible de générer une erreur
} catch (error) {
    console.error("Erreur :", error.message);
} finally {
    console.log("Cette partie s'exécute toujours");
}

2.5.3. Throw

L'instruction throw permet de générer des erreurs personnalisées : 

function diviser(a, b) {
    if (b === 0) {
        throw new Error("Division par zéro impossible");
    }
    return a / b;
}

try {
    console.log(diviser(10, 0));
} catch (error) {
    console.error("Erreur de division :", error.message);
}

2.5.4. Types d'erreurs

JavaScript possède plusieurs types d'erreurs prédéfinis :

  • Error : l'erreur de base
  • SyntaxError : erreur de syntaxe
  • ReferenceError : référence à une variable non définie
  • TypeError : opération sur un type incorrect
  • RangeError : valeur hors de la plage autorisée

2.5.5. Gestion des erreurs asynchrones

Pour les opérations asynchrones, on peut utiliser .catch() avec les promesses ou try...catch avec async/await : 

// Avec les promesses
fetchData()
    .then(data => console.log(data))
    .catch(error => console.error("Erreur :", error));

// Avec async/await
async function getData() {
    try {
        const data = await fetchData();
        console.log(data);
    } catch (error) {
        console.error("Erreur :", error);
    }
}

2.5.6. Erreurs personnalisées

On peut créer des classes d'erreurs personnalisées pour une meilleure gestion : 

class ValidationError extends Error {
    constructor(message) {
        super(message);
        this.name = "ValidationError";
    }
}

function validateUser(user) {
    if (!user.name) {
        throw new ValidationError("Le nom de l'utilisateur est requis");
    }
}

try {
    validateUser({});
} catch (error) {
    if (error instanceof ValidationError) {
        console.error("Erreur de validation :", error.message);
    } else {
        console.error("Erreur inattendue :", error);
    }
}

2.5.7. Bonnes pratiques

  • Toujours inclure la gestion des erreurs dans le code asynchrone
  • Utiliser des erreurs spécifiques pour faciliter le débogage
  • Loguer les erreurs de manière appropriée (console, service de logging)
  • Ne pas exposer les détails sensibles des erreurs aux utilisateurs
  • Tester la gestion des erreurs dans le cadre des tests unitaires

Une bonne gestion des erreurs améliore la robustesse et la maintenabilité du code JavaScript.

3. DOM et manipulation du navigateur

3.1. Sélection et manipulation des éléments DOM

La sélection et la manipulation des éléments DOM (Document Object Model) sont des compétences essentielles en JavaScript pour interagir avec les pages web. Voici un résumé détaillé :

3.1.1. Sélection des éléments DOM

Il existe plusieurs méthodes pour sélectionner des éléments DOM :

  • document.getElementById('id') : Sélectionne un élément par son ID
  • document.getElementsByClassName('class') : Sélectionne des éléments par leur classe
  • document.getElementsByTagName('tag') : Sélectionne des éléments par leur balise
  • document.querySelector('selector') : Sélectionne le premier élément correspondant au sélecteur CSS
  • document.querySelectorAll('selector') : Sélectionne tous les éléments correspondant au sélecteur CSS

Exemples :

// Sélectionner par ID
const header = document.getElementById('main-header');

// Sélectionner par classe
const paragraphs = document.getElementsByClassName('content');

// Sélectionner par balise
const buttons = document.getElementsByTagName('button');

// Utiliser querySelector
const firstLink = document.querySelector('a');

// Utiliser querySelectorAll
const allLinks = document.querySelectorAll('a');

3.1.2. Manipulation des éléments DOM

Une fois les éléments sélectionnés, on peut les manipuler de diverses façons :

3.1.2.1. Modifier le contenu 

// Modifier le texte
element.textContent = 'Nouveau texte';

// Modifier le HTML
element.innerHTML = '<strong>Nouveau contenu HTML</strong>';

3.1.2.2. Modifier les attributs 

// Définir un attribut
element.setAttribute('class', 'highlight');

// Obtenir un attribut
const href = element.getAttribute('href');

// Supprimer un attribut
element.removeAttribute('disabled');

3.1.2.3. Modifier les styles 

// Modifier directement le style
element.style.color = 'red';
element.style.fontSize = '16px';

// Ajouter ou supprimer des classes
element.classList.add('active');
element.classList.remove('hidden');
element.classList.toggle('selected');

3.1.2.4. Créer et supprimer des éléments 

// Créer un nouvel élément
const newDiv = document.createElement('div');
newDiv.textContent = 'Nouveau div';

// Ajouter l'élément au DOM
document.body.appendChild(newDiv);

// Supprimer un élément
const oldElement = document.getElementById('old-element');
oldElement.parentNode.removeChild(oldElement);

3.1.2.5. Parcourir le DOM 

// Accéder aux éléments parents
const parent = element.parentNode;

// Accéder aux éléments enfants
const children = element.children;
const firstChild = element.firstElementChild;
const lastChild = element.lastElementChild;

// Accéder aux éléments frères
const nextSibling = element.nextElementSibling;
const previousSibling = element.previousElementSibling;

En maîtrisant ces techniques de sélection et de manipulation du DOM, vous pouvez créer des pages web dynamiques et interactives en JavaScript.

3.2. Événements et gestion des événements

La gestion des événements est un aspect crucial du développement JavaScript, permettant de créer des interfaces utilisateur interactives. Voici un résumé détaillé avec des exemples :

3.2.1. Types d'événements courants

  • Événements de souris : click, dblclick, mouseover, mouseout
  • Événements de clavier : keydown, keyup, keypress
  • Événements de formulaire : submit, change, focus, blur
  • Événements de fenêtre : load, resize, scroll

3.2.2. Ajout d'écouteurs d'événements

Il existe plusieurs façons d'ajouter des écouteurs d'événements :

3.2.2.1. Méthode addEventListener() 

const button = document.querySelector('#myButton');
button.addEventListener('click', function() {
    console.log('Button clicked!');
});

3.2.2.2. Propriétés on-event 

const button = document.querySelector('#myButton');
button.onclick = function() {
    console.log('Button clicked!');
};

3.2.3. L'objet Event

Chaque gestionnaire d'événements reçoit un objet Event contenant des informations sur l'événement : 

document.addEventListener('click', function(event) {
    console.log('Clicked at coordinates: ' + event.clientX + ', ' + event.clientY);
});

3.2.4. Propagation des événements

Les événements se propagent à travers le DOM en deux phases : capture et bouillonnement.

3.2.4.1. Bouillonnement (Bubbling) 

document.querySelector('#child').addEventListener('click', function() {
    console.log('Child clicked');
});

document.querySelector('#parent').addEventListener('click', function() {
    console.log('Parent clicked');
});

3.2.4.2. Capture 

document.querySelector('#parent').addEventListener('click', function() {
    console.log('Parent clicked (capture phase)');
}, true);

3.2.5. Prévention du comportement par défaut 

document.querySelector('form').addEventListener('submit', function(event) {
    event.preventDefault();
    console.log('Form submission prevented');
});

3.2.6. Délégation d'événements

Utile pour gérer de nombreux éléments similaires : 

document.querySelector('#parent-list').addEventListener('click', function(event) {
    if (event.target.tagName === 'LI') {
        console.log('List item clicked:', event.target.textContent);
    }
});

3.2.7. Suppression d'écouteurs d'événements 

function handleClick() {
    console.log('Clicked');
}

const button = document.querySelector('#myButton');
button.addEventListener('click', handleClick);

// Plus tard
button.removeEventListener('click', handleClick);

3.2.8. Événements personnalisés 

const customEvent = new CustomEvent('myCustomEvent', {
    detail: { message: 'Hello Custom Event!' }
});

document.addEventListener('myCustomEvent', function(e) {
    console.log(e.detail.message);
});

document.dispatchEvent(customEvent);

La maîtrise de ces concepts de gestion des événements permet de créer des interfaces utilisateur réactives et interactives en JavaScript.

3.3. AJAX et Fetch API

AJAX (Asynchronous JavaScript and XML) et l'API Fetch sont des technologies essentielles pour effectuer des requêtes asynchrones en JavaScript. Voici un résumé détaillé de ces deux approches :

3.3.1. AJAX (Asynchronous JavaScript and XML)

AJAX permet d'envoyer et de recevoir des données du serveur de manière asynchrone, sans avoir à recharger la page entière. Bien que son nom contienne "XML", AJAX peut travailler avec divers formats de données, y compris JSON, HTML, et du texte brut.

Exemple d'utilisation d'AJAX avec XMLHttpRequest : 

var xhr = new XMLHttpRequest();
xhr.onreadystatechange = function() {
    if (xhr.readyState === 4 && xhr.status === 200) {
        console.log(xhr.responseText);
    }
};
xhr.open("GET", "https://api.example.com/data", true);
xhr.send();

3.3.2. Fetch API

L'API Fetch est une interface plus moderne pour faire des requêtes HTTP. Elle utilise les Promesses, ce qui la rend plus facile à utiliser et à comprendre que l'ancienne API XMLHttpRequest.

Exemple d'utilisation de Fetch : 

fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Erreur:', error));

Utilisation de Fetch avec async/await : 

async function fetchData() {
    try {
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Erreur:', error);
    }
}

fetchData();

3.3.3. Comparaison entre AJAX et Fetch

  • Fetch est basé sur les Promesses, ce qui le rend plus facile à utiliser avec async/await.
  • Fetch a une syntaxe plus simple et plus concise que XMLHttpRequest.
  • Fetch ne rejette pas la promesse sur les erreurs HTTP (comme 404 ou 500), il faut vérifier le statut manuellement.
  • AJAX via XMLHttpRequest est toujours utile pour la compatibilité avec les anciens navigateurs.

3.3.4. Envoi de données avec Fetch 

fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        name: 'John Doe',
        email: 'john@example.com'
    })
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Erreur:', error));

En conclusion, bien qu'AJAX reste pertinent, notamment pour la compatibilité avec d'anciens systèmes, l'API Fetch est généralement préférée pour sa simplicité et son intégration naturelle avec les fonctionnalités modernes de JavaScript comme les Promesses et async/await.

3.4. Web Storage (localStorage, sessionStorage)

Le Web Storage est une API JavaScript qui permet de stocker des données côté client dans le navigateur. Il existe deux mécanismes principaux : localStorage et sessionStorage. Voici un résumé détaillé avec des exemples :

3.4.1. localStorage

localStorage permet de stocker des données sans date d'expiration. Les données persistent même après la fermeture du navigateur.

Stocker des données : 

localStorage.setItem('username', 'John');
localStorage.setItem('isLoggedIn', true);
localStorage.setItem('userPreferences', JSON.stringify({ theme: 'dark', fontSize: 14 }));

Récupérer des données : 

const username = localStorage.getItem('username');
const isLoggedIn = localStorage.getItem('isLoggedIn') === 'true';
const userPreferences = JSON.parse(localStorage.getItem('userPreferences'));

Supprimer des données : 

localStorage.removeItem('username');
localStorage.clear(); // Supprime toutes les données

3.4.2. sessionStorage

sessionStorage stocke les données pour une seule session. Les données sont effacées lorsque l'onglet ou la fenêtre du navigateur est fermé.

Utilisation (similaire à localStorage) : 

sessionStorage.setItem('tempData', 'Cette donnée sera perdue à la fermeture');
const tempData = sessionStorage.getItem('tempData');
sessionStorage.removeItem('tempData');
sessionStorage.clear();

3.4.3. Différences clés

  • Persistance : localStorage persiste après la fermeture du navigateur, sessionStorage non.
  • Portée : localStorage est partagé entre tous les onglets et fenêtres du même origine, sessionStorage est limité à un seul onglet/fenêtre.

3.4.4. Limites et considérations

  • Capacité de stockage limitée (généralement 5-10 MB)
  • Stockage en chaînes de caractères uniquement (utiliser JSON pour les objets)
  • Pas de cryptage natif (ne pas stocker de données sensibles)

3.4.5. Exemple pratique : Thème utilisateur 

// Sauvegarder le thème
function saveTheme(theme) {
    localStorage.setItem('userTheme', theme);
}

// Récupérer le thème
function getTheme() {
    return localStorage.getItem('userTheme') || 'light'; // 'light' par défaut
}

// Utilisation
saveTheme('dark');
console.log(getTheme()); // Affiche 'dark'

// Appliquer le thème au chargement de la page
document.addEventListener('DOMContentLoaded', () => {
    const currentTheme = getTheme();
    document.body.className = currentTheme;
});

En conclusion, Web Storage offre une solution simple et efficace pour stocker des données côté client, améliorant les performances et l'expérience utilisateur des applications web.
